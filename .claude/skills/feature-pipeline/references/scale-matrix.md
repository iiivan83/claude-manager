# Матрица масштабов — Feature Pipeline

## Критерии классификации

### Мелкая (small)
- Изолированное изменение: правка внутренней логики, исправление бага, добавление простой функции
- Не меняет интерфейсы между модулями
- Затрагивает 1-2 файла
- Не требует новых абстракций

### Средняя (medium)
- Заметное изменение: новая функциональность, изменение API модуля
- Затрагивает взаимодействие между компонентами
- Затрагивает 3-10 файлов
- Может требовать новых тестов на нескольких уровнях

### Крупная (large)
- Структурное изменение: новые модули, изменение архитектуры
- Переработка существующих интерфейсов
- Затрагивает 10+ файлов
- Требует ADR

- Фаза 7 — E2E тесты — ПРОПУСКАЕТСЯ

- Фаза 9 — Документация — ДА
