# Правила для скиллов

Этот файл автоматически подгружается когда Claude работает с файлами в директории скиллов.

## Именование

- Папки скиллов — через дефис, в нижнем регистре
- Название должно максимально точно описывать что скилл ДЕЛАЕТ
- Никаких общих, абстрактных или универсальных названий
- Плохо: `data-tool`, `helper`, `processor`, `manager`
- Хорошо: `extract-budget-metrics`, `archive-expired-budgets`, `create-project-dev-structure`

## Структура скилла

Каждый скилл — это папка со следующим содержимым:

- **SKILL.md** (обязательно) — frontmatter (name, description) + инструкции, до 1000 строк
- **agents/** (по необходимости) — инструкции для подагентов, один файл = один агент
- **scripts/** (по необходимости) — скрипты для повторяющихся/детерминированных задач
- **references/** (по необходимости) — справочные материалы, схемы данных
- **evals/** (по необходимости) — тестовые наборы: `evals.json` по схеме из `~/.claude/references/evals-schema.md`

## SKILL.md — обязательные поля frontmatter

- **name** — идентификатор скилла (совпадает с именем папки)
- **description** — когда триггерить и что делает. Это главный механизм запуска — описание должно быть подробным и включать ключевые слова на русском и английском

## Правило сессионного отчёта

Каждый пайплайн-скилл (скилл, который запускает цепочку агентов или выполняет многошаговую работу) ОБЯЗАН делать сессионный отчёт в конце работы через скилл `session-report`.

## Стиль письма

Каждый скилл и каждый агент ОБЯЗАН следовать единому стилю письма из `~/.claude/references/writing-style-guide.md`.

Перед началом работы прочитай этот файл. Все создаваемые документы (отчёты, спецификации, логи для человека, описания проблем, чек-листы) должны соответствовать этому стандарту.

## Куда что класть

- Спецификации → `dev/docs/specs/`, после реализации переносить в `dev/docs/specs/realised/`
- Логи пайплайнов → `dev/docs/logs/`
- Сессионные отчёты → `dev/docs/session-reports/DD-MM/`
- ADR (архитектурные решения) → `dev/docs/adr/`
- Changelog (журнал изменений) → `dev/docs/changelog/`
- BRD (бизнес-требования, CJM) → `dev/docs/brd/`
- Лог обновлений CLAUDE.md → `dev/docs/claude-md-updates/`
- Временные файлы → `dev/temp/`

## Шаблоны документов

При создании любого документа (ADR, changelog, BRD, CLAUDE.md update) скилл ОБЯЗАН:

1. Прочитать файл `~/.claude/doc-templates.md` и найти секцию с нужным типом документа
2. Заполнить все обязательные атрибуты из шаблона
3. Сохранить документ в правильную папку (см. «Куда что класть»)

## Именование документов

Все документы (кроме BRD и сессионных отчётов) именуются по единому формату:

`{DD.MM_HH.MM}-{skill-name}-{entity-name}.md`

Где:
- **DD.MM_HH.MM** — дата и время создания (например, `03.04_14.30`)
- **skill-name** — скилл, который создал документ
- **entity-name** — сущность, которую документ описывает

Исключения:
- **Сессионные отчёты** → `{ЧЧ-МИН}_{краткое-название}.md` в подпапке `DD-MM/`
- **BRD** → `brd-{название-темы}.md`
- **CLAUDE.md updates** → `{DD.MM_HH.MM}-{skill-name}.md` (без entity, потому что сущность всегда одна — CLAUDE.md)
