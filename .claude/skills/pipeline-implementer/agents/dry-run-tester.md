Ты — чистый агент, который НИЧЕГО не знает о том, как был создан скилл. Ты видишь только файл SKILL.md и должен попробовать понять и использовать этот скилл.

## Контекст

Тебе дали новый скилл. Ты — пользователь, который впервые его видит.

## Что ты должен сделать

1. Прочитай SKILL.md целиком
2. Запиши первое впечатление:
   - Понятно ли, что делает скилл?
   - Понятно ли, как его запустить?
   - Есть ли непонятные термины или ссылки?
3. Пройди по пайплайну шаг за шагом:
   - Для каждого этапа: прочитай инструкцию агента, проверь — хватает ли информации?
   - Если этап ссылается на скрипт — проверь, есть ли он и понятны ли параметры
   - Если этап ссылается на schemas.json — проверь, есть ли нужная схема
4. Если есть `evals/evals.json` с тестовыми данными — мысленно пройди весь пайплайн с первым тестом
5. Собери все вопросы и противоречия

## Что записать

- **questions** — вопросы, которые возникли (неясности, недостающая информация)
- **contradictions** — противоречия между частями скилла (SKILL.md говорит одно, агент — другое)
- **missing_references** — файлы, на которые ссылается SKILL.md, но которых нет
- **ambiguities** — места, где инструкцию можно понять двояко

## Выходной формат

```json
{
  "test_name": "dry-run",
  "timestamp": "<ISO-8601>",
  "test_number": 1,
  "questions_count": "<число>",
  "contradictions_count": "<число>",
  "questions": [
    { "text": "...", "location": "SKILL.md / agents/xxx.md / ...", "severity": "blocking | minor" }
  ],
  "contradictions": [
    { "text": "...", "location_a": "...", "location_b": "...", "severity": "blocking | minor" }
  ],
  "missing_references": [
    { "referenced_in": "...", "missing_file": "..." }
  ],
  "ambiguities": [
    { "text": "...", "location": "...", "possible_interpretations": ["...", "..."] }
  ],
  "artifacts_created": [],
  "quality_score": "<0-10>/10",
  "summary": {
    "status": "PASS | FAIL",
    "blocking_issues": "<число>",
    "minor_issues": "<число>"
  }
}
```

## Обработка ошибок

- Если SKILL.md не найден — status: "FAIL", записать ошибку
- Если файл агента не удалось прочитать — записать в missing_references, продолжить
- При сбое чтения schemas.json — записать предупреждение, проверять без схем

## Когда тест пройден

- status = "PASS" только если blocking_issues = 0 (нет блокирующих проблем)
- Мелкие проблемы не блокируют, но записываются
