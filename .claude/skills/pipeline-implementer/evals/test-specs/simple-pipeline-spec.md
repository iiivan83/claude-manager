# Спецификация пайплайна: csv-report-generator

Простой пайплайн для обработки CSV-файлов и генерации отчётов. Принимает путь к CSV-файлу, парсит данные, считает базовую статистику и создаёт JSON-отчёт.

## Общие сведения

- **Название пайплайна:** csv-report-generator
- **Назначение:** автоматизация рутинного анализа CSV — вместо ручного открытия файла и подсчёта статистики пайплайн делает всё сам
- **Входные данные:** путь к CSV-файлу
- **Выходные данные:** JSON-отчёт со статистикой

## Этапы

### Этап 0 — Подготовка (скрипт)

- **Тип:** скрипт (bash)
- **Что делает:** создаёт рабочие папки (output/, logs/), проверяет что CSV-файл существует и читается
- **Вход:** путь к CSV-файлу, путь к рабочей директории
- **Выход:** подготовленная структура папок, валидация входного файла
- **Зависимости:** нет

### Этап 1 — Парсинг данных (агент)

- **Тип:** агент
- **Что делает:** читает CSV-файл, определяет типы столбцов (числовые, текстовые, даты), проверяет целостность данных (пустые ячейки, некорректные значения), создаёт структурированное представление
- **Вход:** путь к CSV-файлу (из Этапа 0)
- **Выход:** agent-outputs/01-parser.json с метаданными: количество строк, список столбцов с типами, обнаруженные проблемы
- **Зависимости:** Этап 0

### Этап 2 — Генерация отчёта (агент)

- **Тип:** агент
- **Что делает:** на основе распарсенных данных считает статистику (среднее, медиана, мин/макс для числовых столбцов; уникальные значения для текстовых), формирует JSON-отчёт
- **Вход:** agent-outputs/01-parser.json
- **Выход:** output/report.json с полной статистикой + agent-outputs/02-reporter.json
- **Зависимости:** Этап 1

## Карта зависимостей

```
Этап 0 (подготовка) → Этап 1 (парсинг) → Этап 2 (отчёт)
```

Все этапы строго последовательные — каждый зависит от предыдущего.

## Схемы данных

### 01-parser.json
```json
{
  "file_path": "string — путь к исходному CSV",
  "row_count": "integer — количество строк (без заголовка)",
  "columns": [
    {
      "name": "string — имя столбца",
      "type": "string — numeric | text | date",
      "null_count": "integer — количество пустых значений"
    }
  ],
  "issues": ["string — описание проблем с данными"]
}
```

### report.json
```json
{
  "source_file": "string",
  "generated_at": "ISO-8601",
  "row_count": "integer",
  "columns": [
    {
      "name": "string",
      "type": "string",
      "stats": {
        "min": "number | null",
        "max": "number | null",
        "mean": "number | null",
        "median": "number | null",
        "unique_values": "integer"
      }
    }
  ],
  "summary": "string — текстовое описание ключевых находок"
}
```
