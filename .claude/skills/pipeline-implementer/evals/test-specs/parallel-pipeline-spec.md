# Спецификация пайплайна: multi-analyzer

Пайплайн для комплексного анализа текстовых документов. Запускает три независимых анализа параллельно (текстовый, числовой, структурный), потом собирает результаты в единую сводку.

## Общие сведения

- **Название пайплайна:** multi-analyzer
- **Назначение:** многоаспектный анализ документов — вместо одного прохода делаем три параллельных специализированных анализа и потом объединяем
- **Входные данные:** путь к документу (markdown или текст)
- **Выходные данные:** JSON-отчёт с результатами всех трёх анализов и общей сводкой

## Этапы

### Этап 0 — Подготовка (скрипт)

- **Тип:** скрипт (bash)
- **Что делает:** создаёт рабочие папки, валидирует входной файл, определяет формат документа
- **Вход:** путь к документу
- **Выход:** подготовленная структура папок, метаданные файла
- **Зависимости:** нет

### Этап 1 — Текстовый анализ (агент)

- **Тип:** агент
- **Что делает:** анализирует текст документа — считает частотность слов, определяет ключевые темы, оценивает читаемость (средняя длина предложения, сложность лексики), находит повторы и клише
- **Вход:** путь к документу (из Этапа 0)
- **Выход:** agent-outputs/01-text-analysis.json
- **Зависимости:** Этап 0
- **Параллельность:** независим от Этапов 2 и 3, можно запускать одновременно

### Этап 2 — Числовой анализ (агент)

- **Тип:** агент
- **Что делает:** извлекает из документа все числовые данные (цифры, проценты, суммы, даты), проверяет их согласованность (суммы сходятся, проценты дают 100%), находит аномалии
- **Вход:** путь к документу (из Этапа 0)
- **Выход:** agent-outputs/02-numeric-analysis.json
- **Зависимости:** Этап 0
- **Параллельность:** независим от Этапов 1 и 3, можно запускать одновременно

### Этап 3 — Структурный анализ (агент)

- **Тип:** агент
- **Что делает:** анализирует структуру документа — заголовки, разделы, вложенность, наличие обязательных секций, сбалансированность (нет ли разделов-гигантов рядом с разделами-карликами)
- **Вход:** путь к документу (из Этапа 0)
- **Выход:** agent-outputs/03-structural-analysis.json
- **Зависимости:** Этап 0
- **Параллельность:** независим от Этапов 1 и 2, можно запускать одновременно

### Этап 4 — Сводный отчёт (агент)

- **Тип:** агент
- **Что делает:** собирает результаты трёх анализов, ищет перекрёстные закономерности (например, сложный текст в плохо структурированных разделах), формирует итоговый отчёт с рекомендациями
- **Вход:** agent-outputs/01-text-analysis.json, agent-outputs/02-numeric-analysis.json, agent-outputs/03-structural-analysis.json
- **Выход:** output/report.json + agent-outputs/04-summary.json
- **Зависимости:** Этапы 1, 2, 3 (все три должны завершиться)

## Карта зависимостей

```
Этап 0 (подготовка)
  ├── Этап 1 (текстовый анализ)  ─┐
  ├── Этап 2 (числовой анализ)   ─┼── Этап 4 (сводка)
  └── Этап 3 (структурный анализ) ─┘
```

Этапы 1, 2, 3 — параллельные, не зависят друг от друга. Этап 4 ждёт завершения всех трёх.

## Схемы данных

### 01-text-analysis.json
```json
{
  "word_count": "integer",
  "avg_sentence_length": "number",
  "readability_score": "number — 0.0-1.0",
  "key_topics": ["string"],
  "repetitions": [{"phrase": "string", "count": "integer"}],
  "cliches": ["string"]
}
```

### 02-numeric-analysis.json
```json
{
  "numbers_found": "integer",
  "percentages": [{"value": "number", "context": "string"}],
  "sums_consistent": "boolean",
  "anomalies": [{"value": "string", "reason": "string", "location": "string"}]
}
```

### 03-structural-analysis.json
```json
{
  "sections_count": "integer",
  "max_depth": "integer",
  "balance_score": "number — 0.0-1.0",
  "missing_sections": ["string"],
  "oversized_sections": [{"name": "string", "line_count": "integer"}]
}
```

### report.json
```json
{
  "source_file": "string",
  "generated_at": "ISO-8601",
  "text_analysis": "object — из 01-text-analysis.json",
  "numeric_analysis": "object — из 02-numeric-analysis.json",
  "structural_analysis": "object — из 03-structural-analysis.json",
  "cross_findings": [{"finding": "string", "sources": ["string"], "severity": "string"}],
  "recommendations": ["string"],
  "overall_quality_score": "number — 0.0-1.0"
}
```
