# Обновление сессионного отчёта

Ты дополняешь существующий сессионный отчёт результатами текущей сессии.
Это решает проблему: по одной теме (например, браузерное тестирование) может быть
10-20 сессий, и вместо 10-20 отдельных файлов удобнее иметь один документ,
в котором вся история работы над темой — от первого обсуждения до финальной реализации.

Единственный источник новых данных — **контекст текущего диалога**. Ты помнишь все
сообщения, все tool calls, все изменения файлов, все обсуждения. Извлекай информацию
оттуда. Не запускай git-команды для сбора данных — всё уже есть в твоей памяти диалога.

Аргументы пользователя (если есть): $ARGUMENTS

## Шаг 1: Найди существующий отчёт

Отчёт для обновления ищи в таком порядке приоритета:

1. **IDE selection** — если пользователь выделил текст в IDE, проверь, содержит ли он
   путь к файлу отчёта (`.md` в `session-reports/`). Если да — используй этот файл.

2. **Аргументы** — если пользователь указал путь к файлу или название отчёта в
   аргументах ($ARGUMENTS), найди этот файл.

3. **Спроси пользователя** — если отчёт не найден ни в selection, ни в аргументах,
   покажи список последних отчётов и попроси выбрать:
   ```bash
   ls -t development/docs/session-reports/2*//*.md 2>/dev/null | head -15
   ```

Прочитай найденный отчёт целиком через Read.

## Шаг 2: Извлеки факты из текущего диалога

Пройдись по всему диалогу и выдели:

1. **Файлы** — какие файлы создавались, редактировались, удалялись
2. **Что именно менялось** — конкретные изменения в каждом файле
3. **Коммиты** — хэши и сообщения (если были)
4. **Решения** — что обсуждалось, какие варианты рассматривались, что выбрали и почему
5. **Проблемы** — что не получилось, какие ошибки возникали, как решились
6. **Незавершённое** — о чём говорили но не сделали, что отложили на потом

## Шаг 3: Сформируй новую секцию

### Получение реального времени (ОБЯЗАТЕЛЬНО)

Claude не знает текущее время — его нужно узнать через системную команду:

```bash
date +"%d-%m-%Y %H:%M"
```

### Формат новой секции

Добавь в конец документа новую секцию по этому шаблону.
Пустые подсекции пропускай — не пиши подсекцию если в ней нечего написать.

```markdown

---

## Сессия DD-MM-YYYY HH:MM

### Резюме

1-2 предложения: что сделано в этой сессии.

### Изменённые файлы

| Файл | Действие | Что сделано |
|------|----------|-------------|
| `путь/к/файлу` | создан/изменён/удалён | конкретное описание |

### Коммиты

- `hash` — сообщение коммита

### Решения

- **Решение**: описание. **Причина**: почему так решили.

### Проблемы и решения

- **Проблема**: описание. **Решение**: как решили.
```

### Правила формирования

- Стиль новой секции должен соответствовать стилю существующего отчёта — если в
  оригинале используются определённые форматы или обозначения, повторяй их
- Никакой воды — каждая строка несёт новый факт (отчёт читает Claude, не человек)
- Конкретные пути, имена, числа — без «некоторые файлы» или «были внесены изменения»
- Не дублируй информацию, которая уже есть в предыдущих секциях отчёта

## Шаг 4: Обнови секцию «Следующие шаги» / «TODO» / «Незавершённое»

Это важный шаг — секция с будущими задачами должна всегда отражать актуальное состояние.

1. Найди в отчёте секцию с задачами — она может называться «Следующие шаги»,
   «TODO», «Незавершённое», «Контекст для следующей сессии» или похоже
2. Если такой секции нет — создай её в конце новой секции сессии
3. Если секция есть:
   - Пункты, которые были выполнены в текущей сессии → пометь как `[x]` (выполнено)
   - Пункты, которые всё ещё актуальны → оставь как есть `[ ]`
   - Новые задачи, которые появились → добавь в конец списка
   - Пункты, которые стали неактуальны → пометь как `[~]` (неактуально) с пояснением

Обновлённую секцию TODO размести **в самом конце документа** (после новой секции сессии),
чтобы при чтении отчёта в следующей сессии актуальные задачи были сразу видны.

## Шаг 5: Запиши изменения

Главное правило: **никогда не переписывай и не удаляй существующее содержимое отчёта**.

Используй Edit, чтобы добавить новую секцию в конец файла. Если в файле была
секция TODO/Следующие шаги — обнови её на месте (через Edit с заменой старой секции
на обновлённую).

## Шаг 6: Итог

Кратко сообщи пользователю:
```
Отчёт обновлён: путь/к/файлу.md
Добавлена секция: Сессия DD-MM-YYYY HH:MM
```
