## CJM-10: Живой стриминг промежуточных обновлений через watcher

### CJM-10-01: Живой стриминг через watcher при работе с инструментами Claude
- **Предусловие:** пользователь подключён к сессии (продолжаем после CJM-02-01), бот запущен через LaunchAgent или nohup, session_watcher активен
- **Шаги:**
  1. Очистить error.log бота (`: > ~/Library/Logs/claude-manager.error.log`), чтобы видеть только новые warning от этого прогона
  2. Отправить боту задачу, требующую вызова инструментов: «Прочитай файл src/claude_manager/bot.py инструментом Read и скажи одним предложением сколько в нём строк кода. Не пиши код, только число.»
  3. Дождаться полного ответа Claude (таймаут 120 секунд — Claude реально читает файл)
  4. После получения финального ответа проверить error.log бота на отсутствие предупреждения «Папка сессий не найдена» для пути `-Users-ivan-Desktop-claude-sandbox-claude-manager`
- **Ожидаемый результат:**
  - Бот отвечает с заголовком вида `#N ✅` и цифрой количества строк в файле
  - В `~/Library/Logs/claude-manager.error.log` **отсутствуют** записи «Папка сессий не найдена» за время прогона сценария
  - Ответ пришёл за разумное время (<120 сек), что косвенно подтверждает работу полной цепочки
- **Способ проверки:**
  - LLM проверяет, что финальный ответ содержит `#N ✅` и число строк
  - Скрипт проверяет error.log через `grep 'Папка сессий не найдена' ~/Library/Logs/claude-manager.error.log` — вывод должен быть пустым
  - Если warning найден — фиксируется регрессия R4 из root-cause отчёта от 10-04-2026, сценарий FAIL
- **Что этот сценарий ловит:** класс багов «контракт с внешним CLI». Проверяет что функция `_encode_project_path` в `src/claude_manager/session_reader.py` воспроизводит реальный алгоритм `sanitizePath()` из Claude CLI, и что session_watcher находит правильную папку сессий на диске. Именно этот класс багов сломал живой мониторинг 10-04-2026 и был починен коммитом afaaf45
