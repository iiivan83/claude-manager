## CJM-04: Создание новой сессии

### CJM-04-01: Создание сессии через /new
- **Предусловие:** бот запущен, пользователь авторизован
- **Шаги:**
  1. Отправить команду `/new`
- **Ожидаемый результат:** бот отвечает подтверждением с номером сессии (например, "Создана новая сессия #N")
- **Способ проверки:** LLM проверяет, что ответ содержит подтверждение создания сессии и номер

## CJM-02: Текстовое сообщение

### CJM-02-01: Обычное текстовое сообщение с ответом
- **Предусловие:** пользователь подключён к сессии (после CJM-04-01)
- **Шаги:**
  1. Отправить текстовое сообщение "Скажи одним словом: какой язык программирования используется в этом проекте?"
- **Ожидаемый результат:** бот отвечает сообщением от Claude с номером сессии и галочкой в заголовке. Ответ содержит "Python"
- **Способ проверки:** LLM проверяет наличие номера сессии с галочкой и осмысленный ответ про Python

### CJM-02-02: Сообщение без активной сессии
- **Предусловие:** пользователь отключён от сессии (после /all)
- **Шаги:**
  1. Отправить команду `/all`
  2. Отправить текстовое сообщение "привет"
- **Ожидаемый результат:** бот отвечает подсказкой о необходимости подключиться к сессии
- **Способ проверки:** LLM проверяет, что ответ содержит указание создать/выбрать сессию

## CJM-05: Просмотр списка сессий

### CJM-05-01: Список сессий после создания
- **Предусловие:** есть хотя бы одна сессия (после CJM-02-01)
- **Шаги:**
  1. Отправить команду `/sessions`
- **Ожидаемый результат:** список сессий с номерами (/1, /2...) и превью первых сообщений
- **Способ проверки:** LLM проверяет наличие пронумерованных сессий в формате /N с превью

## CJM-06: Переключение на сессию

### CJM-06-01: Переключение на существующую сессию по номеру
- **Предусловие:** есть хотя бы одна сессия в списке (после CJM-05-01)
- **Шаги:**
  1. Отправить команду с номером сессии из списка (например `/1`)
- **Ожидаемый результат:** бот подтверждает подключение к сессии с номером и превью
- **Способ проверки:** LLM проверяет, что ответ содержит подтверждение подключения к конкретной сессии

### CJM-06-02: Переключение на несуществующую сессию
- **Предусловие:** бот запущен
- **Шаги:**
  1. Отправить команду `/99`
- **Ожидаемый результат:** бот сообщает об ошибке — сессия не найдена
- **Способ проверки:** LLM проверяет, что ответ содержит сообщение об ошибке/не найдена

## CJM-08: Остановка Claude

### CJM-08-01: Остановка через /stop в сессии
- **Предусловие:** пользователь подключён к сессии
- **Шаги:**
  1. Сначала подключиться к сессии через /new
  2. Отправить команду `/stop`
- **Ожидаемый результат:** бот подтверждает остановку Claude или сообщает, что нечего останавливать
- **Способ проверки:** LLM проверяет, что ответ содержит подтверждение остановки или информацию об отсутствии активного процесса

### CJM-08-02: /stop без активной сессии (в режиме /all)
- **Предусловие:** пользователь в режиме мониторинга
- **Шаги:**
  1. Отправить команду `/all`
  2. Отправить команду `/stop`
- **Ожидаемый результат:** бот сообщает, что команда /stop работает только внутри сессии
- **Способ проверки:** LLM проверяет, что ответ указывает на необходимость быть в сессии
