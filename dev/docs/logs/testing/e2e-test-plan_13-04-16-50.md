## Порядок тестирования

## CJM-04: Создание новой сессии

### CJM-04-01: Создание сессии через /new
- **Предусловие:** бот запущен, пользователь не подключён к сессии
- **Шаги:**
  1. Отправить команду `/new`
- **Ожидаемый результат:** бот отвечает подтверждением с номером сессии (например, «Создана новая сессия #1»)
- **Способ проверки:** LLM проверяет, что ответ содержит подтверждение создания сессии и дневной номер

---

## CJM-02: Отправка текстового сообщения

### CJM-02-01: Обычное текстовое сообщение с ответом
- **Предусловие:** пользователь подключён к сессии (после CJM-04-01)
- **Шаги:**
  1. Отправить сообщение: `Скажи одним предложением: какой сегодня день недели?`
- **Ожидаемый результат:** бот отвечает текстом Claude с номером сессии и галочкой (формат `#N ✅`)
- **Способ проверки:** LLM проверяет наличие осмысленного ответа и заголовка с номером сессии

### CJM-02-02: Сообщение без активной сессии
- **Предусловие:** пользователь в режиме /all (не подключён к сессии)
- **Шаги:**
  1. Отправить команду `/all`
  2. Отправить текстовое сообщение: `Привет`
- **Ожидаемый результат:** бот отвечает подсказкой — создать сессию через /new или подключиться через /sessions
- **Способ проверки:** LLM проверяет, что ответ содержит предложение создать/выбрать сессию

## CJM-10: Живой стриминг промежуточных обновлений

### CJM-10-01: Промежуточные обновления приходят до финального ответа
- **Предусловие:** пользователь подключён к сессии
- **Шаги:**
  1. Подключиться к сессии (если ещё не подключён) через номер из предыдущего теста
  2. Отправить сообщение, требующее заметной работы: `Прочитай файл src/claude_manager/bot.py и перечисли все команды, которые он обрабатывает. Список команд.`
- **Ожидаемый результат:** перед финальным ответом (#N ✅) приходит хотя бы одно промежуточное обновление (#N ⏳)
- **Способ проверки:** проверяем, что среди ответов есть сообщение с ⏳ ДО сообщения с ✅

## CJM-08: Остановка Claude

### CJM-08-01: Команда /stop в сессии
- **Предусловие:** пользователь подключён к сессии
- **Шаги:**
  1. Подключиться к сессии (отправить номер сессии)
  2. Отправить длинную задачу: `Прочитай все файлы в src/claude_manager/ один за другим и для каждого напиши подробное описание каждой функции`
  3. Через 3 секунды отправить `/stop`
- **Ожидаемый результат:** бот подтверждает остановку Claude
- **Способ проверки:** LLM проверяет подтверждение остановки

### CJM-08-02: Команда /stop без сессии (режим /all)
- **Предусловие:** пользователь в режиме /all
- **Шаги:**
  1. Отправить `/all`
  2. Отправить `/stop`
- **Ожидаемый результат:** бот сообщает, что /stop работает только внутри сессии
- **Способ проверки:** LLM проверяет наличие сообщения о необходимости быть в сессии

## CJM-11: Переключение между проектами

### CJM-11-01: Список проектов через /projects
- **Предусловие:** бот запущен, PROJECTS_ROOT_DIR содержит проекты
- **Шаги:**
  1. Отправить команду `/projects`
- **Ожидаемый результат:** бот отвечает списком проектов с кликабельными /p1, /p2... и маркером ● у текущего
- **Способ проверки:** LLM проверяет наличие списка проектов с маркером текущего

### CJM-11-02: Переключение на другой проект
- **Предусловие:** есть минимум 2 проекта в PROJECTS_ROOT_DIR
- **Шаги:**
  1. Отправить `/projects` — запомнить номер НЕтекущего проекта
  2. Отправить `/pN` (номер другого проекта)
- **Ожидаемый результат:** бот подтверждает переключение с именем проекта
- **Способ проверки:** LLM проверяет подтверждение переключения

### CJM-11-03: Возврат в исходный проект
- **Предусловие:** пользователь переключился на другой проект (CJM-11-02)
- **Шаги:**
  1. Отправить `/projects`
  2. Переключиться обратно на исходный проект (claude_manager)
- **Ожидаемый результат:** бот подтверждает переключение обратно
- **Способ проверки:** LLM проверяет подтверждение и что текущий маркер ● сместился
