## Расширенные сценарии (цепочки действий)

Длинные сценарии, которые проверяют реальные пользовательские потоки — несколько команд подряд, переключения между сессиями и режимами.

---

### FLOW-01: Полный жизненный цикл сессии

Пользователь создаёт сессию, общается с Claude, уходит в мониторинг, возвращается обратно.

**Шаги:**

1. `/new` → бот отвечает «Создана новая сессия #N»
   - **Проверка:** ответ содержит «Создана новая сессия» и номер #N
   - **Запомнить:** номер N для дальнейших шагов

2. Отправить: «Скажи одним словом: привет» → Claude отвечает
   - **Проверка:** ответ начинается с `#N ✅` (номер текущей сессии, галочка — значит финальный ответ)
   - **Проверка:** в ответе есть слово, похожее на приветствие

3. `/all` → бот отвечает «Режим мониторинга всех сессий»
   - **Проверка:** ответ содержит «Режим мониторинга»

4. Отправить: «Привет» → бот НЕ отправляет в Claude
   - **Проверка:** ответ содержит подсказку про /new или /sessions (потому что в режиме мониторинга нельзя писать)

5. `/sessions` → бот показывает список сессий
   - **Проверка:** в списке есть `/N` (наша сессия) с превью текста

6. `/N` (номер из шага 1) → бот подключает к сессии
   - **Проверка:** ответ содержит «Подключён к сессии #N»

7. Отправить: «Сколько будет 2+2? Ответь одним числом» → Claude отвечает
   - **Проверка:** ответ начинается с `#N ✅`
   - **Проверка:** ответ содержит «4»

---

### FLOW-02: Две сессии и навигация между ними

Пользователь работает в двух сессиях и переключается между ними.

**Шаги:**

1. `/new` → «Создана новая сессия #A»
   - **Запомнить:** номер A

2. Отправить: «Запомни слово: яблоко. Больше ничего не отвечай, просто скажи ок» → Claude отвечает
   - **Проверка:** ответ с заголовком `#A ✅`

3. `/new` → «Создана новая сессия #B»
   - **Запомнить:** номер B
   - **Проверка:** B > A (новая сессия получила следующий номер)

4. Отправить: «Запомни слово: банан. Больше ничего не отвечай, просто скажи ок» → Claude отвечает
   - **Проверка:** ответ с заголовком `#B ✅` (мы в сессии B)

5. `/sessions` → список сессий
   - **Проверка:** в списке есть и `/A`, и `/B`

6. `/A` → подключение к первой сессии
   - **Проверка:** ответ содержит «Подключён к сессии #A»

7. Отправить: «Какое слово я просил тебя запомнить?» → Claude отвечает
   - **Проверка:** ответ с `#A ✅`
   - **Проверка:** ответ содержит «яблоко» (Claude помнит контекст первой сессии)

8. `/B` → подключение ко второй сессии
   - **Проверка:** ответ содержит «Подключён к сессии #B»

9. Отправить: «Какое слово я просил тебя запомнить?» → Claude отвечает
   - **Проверка:** ответ с `#B ✅`
   - **Проверка:** ответ содержит «банан» (контекст второй сессии отдельный от первой)

### FLOW-03: Ошибки и ограничения

Пользователь пробует действия, которые должны быть заблокированы — бот вежливо подсказывает.

**Шаги:**

1. `/all` → «Режим мониторинга»

2. Отправить: «Привет» → бот блокирует
   - **Проверка:** ответ содержит подсказку про /new или /sessions

3. `/stop` → бот блокирует
   - **Проверка:** ответ содержит «работает только внутри сессии»

4. `/99` → бот не находит сессию
   - **Проверка:** ответ содержит «не найдена» или номер 99

5. `/new` → «Создана новая сессия #N»
   - **Проверка:** сессия успешно создана

6. `/stop` → бот сообщает, что Claude не работает
   - **Проверка:** ответ содержит «не работает» или «нечего останавливать» (Claude не запущен, только сессия создана)

### FLOW-04: Мониторинг — ссылки и заголовки

Проверяет, что ответы из текущей сессии приходят с `#N` (без ссылки), а ответы из других сессий — с кликабельным `/N`.

**Контекст:** бот оформляет номера сессий по-разному:
- Текущая сессия → `#N ✅` или `#N ⏳` (просто текст, номер со знаком #)
- Другая сессия → `/N ✅` или `/N ⏳` (кликабельная команда, номер со знаком /)

**Шаги:**

1. `/new` → «Создана новая сессия #A»
   - **Запомнить:** номер A

2. Отправить: «Напиши ровно три слова: раз два три» → Claude отвечает
   - **Проверка формата:** ответ начинается с `#A` (знак #, не /) — потому что мы ВНУТРИ этой сессии
   - **Проверка содержания:** ответ содержит «раз два три»

3. `/new` → «Создана новая сессия #B»
   - **Запомнить:** номер B

4. Отправить длинный запрос, на который Claude будет думать несколько секунд: «Напиши пошаговый план того, как приготовить борщ. Подробно, не менее 10 шагов»
   - **Не ждём полный ответ** — сразу переходим к шагу 5

5. **Пока Claude думает над сессией B** — ждём любое сообщение от бота (до 120 сек)
   - Если приходит промежуточное обновление (#B ⏳) — это тоже ок, ждём финальный
   - **Проверка финального ответа:** начинается с `#B` (знак #) — мы всё ещё в сессии B

6. `/A` → подключение к первой сессии

7. `/new` → создаём сессию #C, отправляем длинный запрос
   - Пока Claude думает в C — ответы из C приходят с `#C` (текущая)
   - Если в это время придёт сообщение из сессии A или B через watcher — оно будет с `/A` или `/B` (со знаком /, кликабельная ссылка)
   - **Проверка:** watcher-сообщения из чужих сессий содержат `/номер` (со слешем), а не `#номер`

**Примечание:** шаги 6-7 зависят от тайминга и того, есть ли активные сессии в терминале. Если watcher-сообщения не приходят — тест проверяет только прямые ответы (шаги 1-5).
