# Сессия 28-03: создание BRD с CJM пользователя

## Резюме

Создан BRD (Business Requirements Document) с полным описанием всех 18 пользовательских путей (Customer Journey Map) для Claude Manager. Документ основан на технической спецификации `development/specs/FULL-TECHNICAL-SPEC.md`.

## Изменённые файлы

| Файл | Действие | Что сделано |
|------|----------|-------------|
| `development/docs/brd-user-journeys.md` | создан | BRD с 18 CJM: все команды бота, очередь сообщений, защита эпохой, режим наблюдения, защита от дублей, сводная карта состояний, сквозные механизмы |
| `development/docs/docs-index.md` | изменён | Добавлен блок для `brd-user-journeys.md` с условиями чтения |

## Решения

- **Формат CJM**: единая структура для всех путей (триггер → предусловия → шаги → ответ системы → результат → edge cases). **Причина**: обеспечивает консистентность и быстрое сканирование.
- **Сводная карта состояний**: добавлена секция с 4 состояниями пользователя (нет сессии / подключён / мониторинг всех / наблюдение). **Причина**: показывает переходы между CJM, которые не очевидны из отдельных путей.
- **Сквозные механизмы**: вынесены в отдельную секцию (проверка доступа, промежуточные сообщения, координация watcher, обновление session_id, отправка в Telegram, жизненный цикл процесса). **Причина**: эти механизмы действуют во всех CJM и их дублирование в каждом пути раздувало бы документ.

## Контекст для следующей сессии

BRD создан и верифицирован. Может использоваться как основа для:
- Написания тестов (каждый CJM = тестовый сценарий)
- Проверки реализации на соответствие требованиям
- Онбординга новых разработчиков (понять все возможности бота)
