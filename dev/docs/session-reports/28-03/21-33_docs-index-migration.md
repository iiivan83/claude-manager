# Сессия 28-03: перенос и адаптация docs-index

## Резюме

Перенесён docs-index.md из проекта `su-main-master 2` в текущий проект `claude_manager`. Сначала по ошибке обновлён оригинал в чужом проекте — откачено. Затем файл скопирован в `development/docs/`, после чего полностью переписан под структуру текущего проекта.

## Изменённые файлы

- **`development/docs/docs-index.md`** — создан. Полностью переписан с нуля: удалено всё содержимое про проект миграции (migration, kubernetes, django, PHP-модули, root-cause-reports, fix-specs и т.д.), вместо этого описаны реальные документы и папки проекта claude_manager
- **`/Users/ivan/Desktop/claude-sandbox/su-main-master 2/migration/docs/docs-index.md`** — изменён по ошибке (добавлены E2E-отчёты, post-migration-audit, session-reports структура), затем полностью откачен до оригинального состояния

## Решения

- **Решение**: docs-index переписать с нуля, а не адаптировать. **Причина**: содержимое из su-main-master 2 полностью не применимо к claude_manager — другой проект, другая структура, другие документы.
- **Решение**: не добавлять в индекс пустые папки как отдельные развёрнутые секции. **Причина**: `script-specs/`, `scripts/`, `temp-docs/` пока пусты — достаточно краткого упоминания.

## Проблемы и решения

- **Проблема**: по ошибке отредактирован оригинал docs-index в `su-main-master 2` (3 правки: E2E-отчёты, post-migration-audit, session-reports). **Решение**: все 3 правки откачены вручную через обратные Edit-операции (файл не отслеживался git, поэтому `git checkout` не сработал).

## Контекст для следующей сессии

Docs-index теперь отражает реальную структуру claude_manager. Документов в проекте пока мало — только CLAUDE.md, FULL-TECHNICAL-SPEC.md и 2 сессионных отчёта. По мере роста проекта индекс нужно обновлять при добавлении новых документов.
