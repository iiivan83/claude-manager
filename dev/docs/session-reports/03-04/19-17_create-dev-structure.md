# Сессия 03-04: создание стандартной dev-структуры проекта

## Резюме

Создана стандартная структура папок `dev/` для артефактов разработки, файл `.claude/skills/AGENTS.md` с правилами для скиллов, скопированы 3 эталонных скилла из `~/.claude/skill-templates/`.

## Изменённые файлы

| Файл | Действие | Что сделано |
|------|----------|-------------|
| `dev/temp/` | создан | Папка для черновиков и временных файлов |
| `dev/docs/session-reports/` | создан | Папка для сессионных отчётов (подпапки по дате DD-MM/) |
| `dev/docs/specs/` | создан | Папка для спецификаций (ТЗ) |
| `dev/docs/specs/realised/` | создан | Папка для реализованных спецификаций |
| `dev/docs/logs/` | создан | Папка для логов пайплайнов и скиллов |
| `dev/scripts/` | создан | Папка для вспомогательных скриптов |
| `.claude/skills/AGENTS.md` | создан | Правила для скиллов: именование, структура, обязательные поля, правило сессионного отчёта, маршрутизация артефактов по папкам |
| `.claude/skills/pipeline-designer/` | скопирован | Эталонный скилл проектирования пайплайнов из `~/.claude/skill-templates/` |
| `.claude/skills/pipeline-explorer/` | скопирован | Эталонный скилл исследования задач из `~/.claude/skill-templates/` |
| `.claude/skills/script-creator-pipeline/` | скопирован | Эталонный скилл сквозного создания скриптов из `~/.claude/skill-templates/` |

## Выполненные команды

- `mkdir -p dev/{temp,docs/{session-reports,specs/realised,logs},scripts}` — создание структуры папок dev/
- `cp -Rn ~/.claude/skill-templates/* .claude/skills/` — копирование эталонных скиллов без перезаписи существующих

## Решения

- **skill-creator не перезаписан**: скилл уже существовал в проекте, флаг `-n` (no-clobber) предотвратил перезапись. **Причина**: сохранение локальных настроек скилла, которые могут отличаться от шаблона.

## Контекст для следующей сессии

Проект теперь имеет две системы хранения артефактов разработки:
- `development/` — старая, используемая существующими скиллами (session-reports, specs, root-cause-reports)
- `dev/` — новая стандартная структура, описанная в AGENTS.md

AGENTS.md маршрутизирует артефакты в `dev/`, но существующие скиллы (session-report, spec-module и др.) по-прежнему пишут в `development/`. Для унификации потребуется либо мигрировать существующие скиллы на `dev/`, либо обновить AGENTS.md чтобы он ссылался на `development/`.
