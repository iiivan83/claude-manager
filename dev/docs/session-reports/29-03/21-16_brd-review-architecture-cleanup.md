# Сессия 29-03: ревью БРД и чистка архитектуры

## Резюме

Ревью БРД на предмет незакрытых технических вопросов. По результатам обсуждения удалены лишние сущности из архитектуры, добавлены недостающие механизмы, полностью перегенерирован документ архитектуры.

## Изменённые файлы

- `development/docs/project_architecture.md` — полная перегенерация. Удалены: /project, /watch, /status, deep link, режим наблюдения (chat_monitor_only), chat_working_dir, EXTERNAL_SESSION_ACTIVITY_THRESHOLD. Добавлены: однопользовательность, обработка невалидного JSON, проверка PID через ps перед убийством процесса.
- `development/docs/brd-user-journeys.md` — добавлена строка «бот рассчитан на одного пользователя», обновлена очистка процессов (проверка имени через ps).
- `development/docs/docs-index.md` — убрано состояние «наблюдение» из описания БРД, исправлено «18 CJM» на «все CJM».

## Решения

- **Однопользовательский бот**: бот не поддерживает нескольких пользователей одновременно. **Причина**: по замыслу проекта — личный пульт управления Claude Code.
- **Удалён режим наблюдения (monitor_only)**: противоречил БРД — в БРД разрешена одновременная работа из Telegram и терминала без блокировок. **Причина**: CJM-02 прямо говорит «писать и из Telegram, и из терминала — в одну сессию».
- **Удалены /project, /watch, /status, deep link**: отсутствуют в БРД, не нужны. **Причина**: всё поведение бота должно быть описано в БРД.
- **Проверка PID через ps**: перед убийством бесхозного процесса проверять что это Claude. **Причина**: за 2 дня ОС может переиспользовать PID для другой программы.
- **Невалидный JSON**: сообщать пользователю + писать в лог. **Причина**: пользователь должен знать о проблемах с Claude CLI.
