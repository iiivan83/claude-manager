# Сессия 14-04: browser-automation-planning

## Резюме

Исследование и выбор инструмента для автоматизации браузера — подписание договоров и задолженностей. Выбран Playwright как Python-библиотека (без MCP-сервера). Установка не завершена — сессия прервана перед `pip install`.

## Решения

- **Решение**: использовать Playwright как Python-библиотеку, без MCP-сервера. **Причина**: пользователю нужен скриптовый подход — один раз написать, запускать многократно. MCP-сервер избыточен для повторяющихся задач.
- **Решение**: вариант 1 — изолированный Chromium от Playwright (не подключение к существующему Chrome через CDP). **Причина**: пользователь выбрал из двух предложенных вариантов.

## Контекст предыдущей сессии (ddf3a678)

В предыдущей сессии обсуждались 5 категорий решений для браузерной автоматизации:
- **Claude in Chrome** — расширение, подключение к Chrome пользователя. Бета, нестабильно.
- **Computer Use** — управление экраном через скриншоты. Медленно, дорого по токенам.
- **Playwright MCP** (Microsoft, 30k+ звёзд) — MCP-сервер, 34 инструмента. Зрелый, стандарт индустрии.
- **PinchTab** — Go-бинарник, CDP-подключение к Chrome. Лёгкий, молодой. Обсудили: сбор данных (`pinchtab_get_text`, `pinchtab_snapshot`, `pinchtab_eval`), скачивание файлов (`pinchtab download`), формы (`pinchtab_fill`, `pinchtab_select`, `pinchtab_click`).
- Другие MCP-серверы (Browserbase, BrowserTools).
