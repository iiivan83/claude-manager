# Сессия 30-03: исправление deep link диалога подтверждения

## Резюме

Проведён root-cause analysis бага: клик по номеру сессии (#N) в watcher-сообщениях показывал диалог «Open tg://msg?text=/N?» вместо мгновенного переключения. Найдена корневая причина (формат `tg://msg?text=` всегда показывает диалог), применены все 4 исправления из отчёта, тесты зелёные, бот перезапущен.

## Изменённые файлы

- `src/claude_manager/bot.py` — изменён — функция `_format_clickable_session_number` (строки 111-113): заменён deep link `<b><a href="tg://msg?text=/N">#N</a></b>` на plain text команду `<b>/N</b>`, которую Telegram автоматически делает кликабельной без диалогов
- `tests/test_bot.py` — изменён — обновлены 2 теста: `test_format_clickable_session_number` (ожидаемый результат `<b>/3</b>`) и `test_send_watcher_message_other_session` (проверка `<b>/1</b>` вместо `tg://msg`)
- `development/specs/realized/bot_spec.md` — изменён — заменён формат deep link на `/N`, добавлено пояснение почему `tg://msg?text=` не используется
- `.claude/skills/review-code/SKILL.md` — изменён — в Pass 4 (BRD-соответствие) добавлен пункт проверки интерактивных элементов

- `development/docs/root-cause-reports/resolved/30-03_06-42_tg-deep-link-confirmation-dialog.md` — создан — полный RCA-отчёт (перемещён в resolved/ после исправления)

## Решения

- **Решение**: заменить `tg://msg?text=/N` deep link на `/N` plain text. **Причина**: `tg://msg?text=` — это Telegram deep link, который ВСЕГДА показывает диалог подтверждения (защита Telegram, обойти нельзя). `/N` как plain text автоматически распознаётся Telegram как команда бота и отправляется мгновенно без диалогов. Этот же подход уже работает в `/sessions`.
- **Решение**: визуальный формат меняется с `#N` на `/N` для кликабельных номеров сессий. **Причина**: BRD не требует именно `#N` для кликабельных ссылок, а `/N` уже используется в `/sessions` и совместим с UX.

## Проблемы и решения

- **Проблема**: после `kill` бота файл `bot.pid` оставался заблокированным, новый процесс не мог стартовать. **Решение**: LaunchAgent автоматически перезапускает бот после kill — не нужно запускать вручную, достаточно убить процесс.
