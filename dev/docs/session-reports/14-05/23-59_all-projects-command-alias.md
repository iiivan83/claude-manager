# Сессия 14-05: команда /all_projects для режима всех проектов

## Коротко

Добавлена команда `/all_projects`, которая включает уже существующий режим мониторинга всех проектов. Это alias для `/all`: старый короткий путь сохранён, новый путь виден в меню команд Telegram и понятнее совпадает с формулировкой "all projects".

## Рабочие файлы

- **`src/claude_manager/bot.py`** — изменён — в список команд Telegram добавлена `/all_projects`; обработчик режима all теперь принимает и `/all`, и `/all_projects`.
- **`tests/test_bot.py`** — изменён — добавлены проверки, что `/all_projects` попадает в меню команд, регистрируется как обработчик и запускает тот же all-mode.
- **`dev/docs/brd/brd-user-journeys.md`** — изменён — пользовательский сценарий мониторинга всех проектов описывает оба входа: `/all` и `/all_projects`.
- **`dev/docs/session-reports/14-05/23-59_all-projects-command-alias.md`** — создан — этот отчёт фиксирует контекст сессии для следующей работы.

## Решения

- **Оставить `/all` и добавить `/all_projects` как alias**: старые пользовательские привычки и существующие сценарии не ломаются, а новая команда появляется в Telegram-меню.
- **Не делать команду `/all projects` с пробелом**: Telegram slash-команды не поддерживают пробелы в имени команды, поэтому выбран допустимый формат с подчёркиванием.
- **Не менять архитектуру all-mode**: режим уже реализован через `all_projects_monitor`, поэтому изменение ограничено маршрутизацией команды и меню подсказок.

## Проверки

- **Точечный red-green цикл** — сначала новые тесты показали отсутствие `/all_projects`, затем после правки прошли.
- **Точечный прогон после правки тестов** — `.venv/bin/python -m pytest tests/test_bot.py::TestHandleAll tests/test_bot.py::TestSetupBot::test_setup_bot_registers_all_projects_command_alias tests/test_bot.py::TestPostInit::test_post_init_sets_commands -q` → `4 passed`.
- **Полный прогон проекта** — `.venv/bin/python -m pytest tests/ -q` → `994 passed, 1 skipped, 3 warnings`.

## Риски и ограничения

- Команда появится в живом Telegram-меню только после штатного перезапуска бота, потому что меню команд выставляется в `post_init` при старте приложения.
- Перезапуск бота из текущей сессии не выполнялся: в проектных правилах указано не запускать restart-скрипт из подпроцесса бота.
- В рабочем дереве до этой задачи уже были unrelated-изменения в `.agents/skills`, `dev/docs/docs-index.md`, старом session report и `docs/superpowers/plans/`; они не относятся к этой сессии и не трогались.

## Продолжение

- После перезапуска бота проверить в Telegram, что `/all_projects` появился в меню подсказок.
- Если пользовательская документация будет отдельно синхронизироваться с реализованными спеками, проверить `dev/docs/specs/realised/bot_spec.md`: он уже был частично устаревшим относительно глобального all-mode и может требовать отдельного обновления.
