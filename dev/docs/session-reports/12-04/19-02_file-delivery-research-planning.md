# Сессия 12-04: исследование способов доставки файлов через Telegram-бота

## Резюме

Исследовали и протестировали 4 способа отправки текстовых файлов (MD, JSON и др.) пользователю через Telegram-бота. Выбран вариант **telegramify-markdown** — библиотека, которая рендерит Markdown прямо в чате Telegram серией форматированных сообщений. Следующий шаг — спецификация инструмента для встраивания в бота.

## Протестированные варианты

### 1. sendDocument (файл как вложение)
- Скрипт `dev/temp/test_send_file.py` — отправлял файл через `bot.send_document()`
- Отправлен тестовый MD-файл, затем реальный файл анализа бюджета Сушкоф (`budgets/sushkof2026/analysis-results/11.04_13.54-budget-analysis.md`, 405 строк, 47 КБ)
- **Результат**: файл приходит, но на телефоне `.md` открывается как plain text без рендеринга

### 2. Telegraph (Instant View)
- Скрипт `dev/temp/test_telegraph.py` — загружал MD на telegra.ph через API, отправлял ссылку
- Написан конвертер MD → Telegraph-ноды (заголовки, списки, код, цитаты, таблицы)
- **Проблема**: Telegraph ограничивает контент страницы ~20 КБ JSON-нод. Файл 47 КБ разбился на 4 части
- **Проблема**: URL-encoding кириллицы раздувает размер в 3 раза (60 КБ JSON → 164 КБ encoded). Решено переключением на JSON Content-Type
- **Результат**: красивый Instant View внутри Telegram, но лимит размера и контент публичный

### 3. GitHub Gist (secret)
- Скрипт `dev/temp/test_gist.py` — создавал secret gist через `gh gist create`, отправлял ссылку
- **Результат**: GitHub рендерит MD красиво, нет лимита размера, можно удалить. Но открывается в браузере, а не внутри Telegram
- Gist удалён после теста: `gh gist delete ee8f160b058f6cedf0d1cc362edf5246`

### 4. telegramify-markdown (рендеринг в чате) — ВЫБРАН
- Скрипт `dev/temp/test_telegramify.py` — использует библиотеку `telegramify-markdown`
- Установлена библиотека: `pip install telegramify-markdown` (зависимости: pyromark, typing-extensions)
- Файл 47 КБ разбился на 7 сообщений (~4000 символов каждое) с форматированием через entities
- **Результат**: контент остаётся внутри Telegram (полная приватность), форматирование читается, нет внешних сервисов

## Выполненные команды

- `pip install telegramify-markdown` — установка библиотеки для конвертации MD → Telegram entities (версия 1.1.2, зависимости: pyromark 0.9.10)

## Решения

- **Выбран telegramify-markdown для доставки файлов**. **Причина**: единственный вариант с полной приватностью (контент не выходит за пределы Telegram) + рендеринг форматирования прямо в чате. Telegraph красивее, но лимит 20 КБ на страницу и публичный доступ. Gist требует браузер. sendDocument не рендерит MD.

## Сравнительная таблица (итог исследования)

- **sendDocument**: приватно, до 50 МБ, но нет рендеринга — plain text на телефоне
- **Telegraph**: красивый Instant View, но лимит ~20 КБ/стр, контент публичный
- **GitHub Gist**: рендеринг GitHub-стиль, нет лимита, но открывается в браузере
- **telegramify-markdown**: рендеринг в чате через entities, приватно, нет внешних зависимостей от сервисов, но длинные документы приходят серией из 5-10 сообщений

## Проблемы и решения

- **Telegraph CONTENT_TOO_BIG**: файл 47 КБ не влезал в одну страницу даже после переключения с URL-encoded на JSON body. Решено разбивкой на чанки по 20 КБ JSON, но это привело к 4 отдельным страницам — неудобно
- **telegramify-markdown API расхождение с документацией**: модуль `telegramify_markdown.type.ContentType` устарел, актуальный — `telegramify_markdown.content` с классами `Text`, `File`, `Photo`. У `Text` — атрибуты `text` (str) и `entities` (list[MessageEntity])
