# Handoff: Codex и Telegram-доставка файлов

## Коротко

В этой сессии разобрали, почему Claude умеет по просьбе пользователя прислать файл или
показать содержимое файла в Telegram, а Codex пока делает это неправильно.

Вывод простой: доставка файлов уже реализована в Telegram-боте. Проблема не в отправке
файлов, а в том, что Codex не знает служебный договор: в финальном ответе нужно вставить
маркер `[SEND_FILE:/absolute/path]` или `[SHOW_FILE:/absolute/path]`.

Правильное продолжение — не переписывать доставку, а научить Codex использовать тот же
механизм, который уже работает для Claude.

## Рабочие файлы

- **`src/claude_manager/file_sender.py`** — утилита, которая парсит маркеры
  `[SEND_FILE:...]` и `[SHOW_FILE:...]`, проверяет тип и размер файла.
- **`src/claude_manager/file_delivery.py`** — слой доставки: по маркеру отправляет файл
  в Telegram как document-вложение или показывает содержимое текстового файла в чате.
- **`src/claude_manager/telegram_response_delivery.py`** — место, где финальный ответ
  агента проходит через обработку файловых маркеров перед отправкой пользователю.
- **`src/claude_manager/claude_code_backend.py`** — backend Claude Code CLI. Уже имеет
  bot-mode инструкцию через `BOT_MODE_SYSTEM_PROMPT_APPENDIX`.
- **`src/claude_manager/codex_backend.py`** — backend Codex CLI. Сейчас собирает команду
  `codex exec ... prompt_text`, но не добавляет отдельное правило про Telegram-доставку
  файлов.
- **`CLAUDE.md`** — проектная инструкция. В разделе "Важные детали для разработки"
  уже описаны маркеры доставки файлов и когда они должны использоваться.
- **`tests/test_codex_backend.py`** — ближайшее место для unit-тестов на формирование
  prompt для Codex.

## Решения

- Доставка файлов должна остаться в существующем контуре `file_sender.py` +
  `file_delivery.py` + `telegram_response_delivery.py`.
- Для Codex нужно добавить короткую bot-mode инструкцию в `codex_backend.py`.
- Инструкция должна объяснять:
  - если пользователь просит "пришли файл", "скинь файл", "отправь файлом" — вставить
    `[SEND_FILE:/absolute/path]`;
  - если пользователь просит "покажи содержимое файла", "что внутри файла" — вставить
    `[SHOW_FILE:/absolute/path]`;
  - путь должен быть абсолютным;
  - не надо показывать пользователю сам маркер как отдельное объяснение: бот вырежет
    маркер и выполнит действие.
- Инструкцию нужно применять и для новой Codex-сессии, и для `resume` существующей
  Codex-сессии.
- Отдельное правило "прислать ссылку в Telegram" сейчас лучше не добавлять как новый
  механизм: обычный URL Telegram и так делает кликабельным. Если позже понадобится
  специальная доставка ссылок, это отдельное требование.

## Проверки

- Прочитан `CLAUDE.md`: в проекте уже зафиксирован контракт маркеров
  `[SEND_FILE:...]` и `[SHOW_FILE:...]`.
- Прочитаны `file_sender.py`, `file_delivery.py`, `telegram_response_delivery.py`:

- Прочитаны `claude_code_backend.py` и `codex_backend.py`: у Claude есть bot-mode
  appendix, у Codex аналогичного правила про файловые маркеры нет.
- Прочитаны тесты вокруг Codex backend и файловой доставки: ближайшая точка для
  регрессии — `tests/test_codex_backend.py`.

## Риски и ограничения

- Код пока не изменялся. Это только расследование и дизайн следующего шага.
- Нельзя чинить проблему только глобальным `~/.codex/AGENTS.md`: тогда правило начнет
  действовать во всех Codex-сессиях, даже вне Telegram-бота.
- Нужно аккуратно проверить фактический CLI-контракт Codex: сейчас prompt передается как
  аргумент команды, поэтому safest path — оборачивать `prompt_text` перед передачей в
  `compose_subprocess_command_args`.
- Live Telegram-проверка не выполнялась. После реализации нужен unit-тест и, если бот
  доступен, ручной smoke через Telegram.
- Секреты не читались и не фиксировались. В отчёт не попали токены, пароли или `.env`.

## Продолжение

1. Написать failing unit-тесты в `tests/test_codex_backend.py`:
   - новая Codex-сессия получает prompt с bot-mode файловыми правилами;
   - `resume` Codex-сессии получает тот же prompt;
   - исходный пользовательский текст сохраняется без потери.
2. В `src/claude_manager/codex_backend.py` добавить константу с инструкцией и маленькую
   функцию, которая добавляет её к пользовательскому prompt.
3. Прогнать целевые тесты:
   - `python -m pytest tests/test_codex_backend.py -q`
   - при необходимости соседние тесты доставки: `python -m pytest tests/test_file_delivery.py tests/test_telegram_response_delivery_behavior.py -q`
4. После зелёных тестов можно сделать короткую ручную проверку через Telegram: выбрать
   Codex, попросить "покажи содержимое файла ..." и убедиться, что пришло сообщение с
   содержимым, а сырой маркер не виден пользователю.
