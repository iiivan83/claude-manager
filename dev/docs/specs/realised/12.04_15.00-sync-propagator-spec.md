### Retry-петли

- **Этап 6B → Этап 5:** если семантическая верификация нашла незакрытые расхождения, пайплайн возвращается к соответствующему под-этапу (5A, 5B или 5C) и повторяет синхронизацию. Счётчик retry — per-sub-stage (каждый под-этап имеет свой счётчик попыток, а не глобальный на весь этап 6). Максимум 2 попытки на под-этап, после — СТОП с эскалацией пользователю
- **Этап 3 (отказ):** если пользователь отклонил план — пайплайн завершается со статусом `user_rejected`, без ошибки

## 2. Общее описание

### Имя пайплайна

`sync-propagator`

### Тип

Claude Code skill (пайплайн-скилл)

### Цель

Обнаружить все расхождения между скиллами, референсами, CLAUDE.md и AGENTS.md в экосистеме проектов — и синхронизировать их по единой цепочке, чтобы изменения из любой точки гарантированно попали во все остальные.

### Проблема, которую решает

Скиллы иногда правятся прямо в целевых проектах (не в budgets), и эти изменения теряются при следующей синхронизации. Бывает и наоборот — в budgets появляются новые принципы или исправления, но до skill-templates и целевых проектов они не доходят неделями. Реальный пример: update-skill в skill-templates отстал от budgets на целый инцидент (10-04), а claude_manager уже получил свежую версию напрямую, минуя templates. Результат — три копии одного скилла в трёх разных состояниях.

### Триггеры запуска

- **Root-cause отчёт** — после apply-root-cause-fixes скиллы могли измениться, нужно пропагировать
- **Запрос пользователя** — явная команда «синхронизируй скиллы»
- **Сессионный отчёт** — если в сессии были изменения скиллов, референсов или CLAUDE.md

### Вход

- **Триггер** (обязательно) — тип: `root_cause_report` (с путём к отчёту), `user_request` (с описанием), `session_report` (с путём к отчёту)
- **Конфигурация** (обязательно) — файл `~/.claude/sync-config.json` со списком проектов и настройками

### Выход

- **sync-report.md** — итоговый человекочитаемый отчёт: что было рассинхронизировано, что синхронизировали, что не удалось
- **orchestrator-log.json** — лог оркестратора со всеми этапами и таймингами
- **sync-backups/** — бэкапы всех изменённых файлов (tar.gz)
- **session-report** — сессионный отчёт по итогам работы

### Рабочие принципы

- **Budgets — единственный первоисточник.** Если скилл изменён в целевом проекте, изменения сначала переносятся в budgets, потом распространяются оттуда. Никогда не наоборот
- **Бэкап перед любым изменением.** Все файлы, которые будут затронуты, сохраняются в tar.gz до начала синхронизации
- **Утверждение пользователем.** План синхронизации показывается пользователю до применения. Без OK — ничего не меняется
- **Checkpoint/resume.** Каждый завершённый под-этап (5A, 5B, 5C) записывается в orchestrator-log. При повторном запуске пропускаются уже завершённые шаги
- **Верификация после каждой синхронизации.** diff -rq подтверждает, что файлы реально совпадают. Отчёт агента «я скопировал» не считается доказательством

### Документооборот

Рабочая директория пайплайна: `dev/docs/logs/sync-propagator/{DD.MM_HH.MM}-sync-propagator/`

Формат `{DD.MM_HH.MM}` — день.месяц_час.минута, например `13.04_15.30`. Соответствует стандарту из `~/.claude/references/document-naming-and-placement.md`.

Внутри:
- `orchestrator-log.json` — лог шагов
- `agent-outputs/` — JSON-результаты каждого агента
- `sync-backups/` — tar.gz бэкапы
- `sync-report.md` — финальный отчёт

### Конфигурация: ~/.claude/sync-config.json

Файл, который определяет всю экосистему проектов. Пользователь редактирует его вручную, пайплайн только читает.

```json
{
  "source": {
    "name": "budgets",
    "path": "/Users/ivan/Desktop/claude-sandbox/budgets",
    "skills_dir": ".claude/skills"
  },
  "skill_templates_path": "/Users/ivan/.claude/skill-templates",
  "references_path": "/Users/ivan/.claude/references",
  "project_specific_skills": ["budget-analyzer", "google-sheets-etl"],
  "excluded_from_templates": ["AGENTS.md"],
  "target_projects": [
    {
      "name": "claude_manager",
      "path": "/Users/ivan/Desktop/claude-sandbox/claude_manager"
    }
  ]
}
```

Поля:
- **source** — проект-первоисточник (budgets). Всегда один
