# Спецификация модуля: session_reader

Дата: 30-03-2026
Слой: 1 (зависит от слоя 0)
Файл: `src/claude_manager/session_reader.py`

## Назначение

Читает файлы сессий Claude Code с диска, извлекает метаданные (идентификатор сессии, время создания, первое сообщение пользователя) и возвращает список последних сессий текущего проекта, отсортированный по времени. Служит единственным источником данных о сессиях для всех модулей, которым нужно показать или найти сессию.

## Обслуживаемые сценарии

- **CJM-05: Просмотр списка сессий (/sessions)** — читает файлы сессий из папки проекта на диске, извлекает из каждого файла session_id, время создания и первое сообщение пользователя (для превью), возвращает 15 последних сессий, отсортированных по времени (новые первые)
- **CJM-06: Переключение на сессию (/N)** — предоставляет данные для поиска сессии по номеру: session_manager и bot используют список сессий от session_reader, чтобы найти нужную сессию среди всех видимых (не только в дневном реестре)
- **CJM-07: Мониторинг всех сессий (/all)** — session_watcher вызывает session_reader каждые 2 секунды для получения актуального списка сессий и отслеживания новых ответов Claude

## Публичный API

### `async def get_recent_sessions(project_dir: str) -> list[SessionInfo]`

Возвращает список последних сессий текущего проекта, отсортированных по времени создания (новые первые).

**Аргументы:**
- `project_dir` (str) — абсолютный путь к рабочей директории проекта (значение `WORKING_DIR` из модуля config). Например: `"/Users/ivan/Desktop/claude-sandbox/claude-manager"`

**Возвращает:** список объектов `SessionInfo` (максимум 15 штук), отсортированных по убыванию времени создания. Если сессий нет — пустой список.

**Исключения:** не выбрасывает. Все ошибки файловой системы ловятся внутри, логируются и возвращается пустой список или список без проблемных сессий.

### `async def get_session_messages(session_id: str, project_dir: str) -> list[dict]`

Читает все сообщения из файла конкретной сессии. Используется session_watcher для отслеживания новых ответов Claude.

**Аргументы:**
- `session_id` (str) — уникальный идентификатор сессии (UUID, например `"eb5ac5bc-2ac6-45ad-8ca9-bb3a1a741f1e"`)
- `project_dir` (str) — абсолютный путь к рабочей директории проекта

**Возвращает:** список словарей — каждый словарь это одна строка из JSONL-файла сессии. Если файл не найден или повреждён — пустой список.

**Исключения:** не выбрасывает. Ошибки логируются, возвращается пустой список.

### `class SessionInfo`

Данные об одной сессии Claude. Используется как структура для передачи между модулями.

**Поля:**
- `session_id` (str) — уникальный идентификатор сессии (UUID). Например: `"eb5ac5bc-2ac6-45ad-8ca9-bb3a1a741f1e"`
- `created_at` (str) — время создания сессии в формате ISO 8601 (UTC). Например: `"2026-03-29T17:23:21.594Z"`
- `preview` (str) — очищенный текст первого сообщения пользователя без обязательной обрезки. Если первое сообщение не найдено — пустая строка `""`

## Внутренние функции

### `def _build_sessions_path(project_dir: str) -> str`

Строит путь к папке, где Claude Code хранит файлы сессий для данного проекта.

**Аргументы:**
- `project_dir` (str) — абсолютный путь к рабочей директории проекта

**Возвращает:** абсолютный путь к папке сессий. Например: `"/Users/ivan/.claude/projects/-Users-ivan-Desktop-claude-sandbox-claude-manager"`

### `def _encode_project_path(project_dir: str) -> str`

Повторяет алгоритм `sanitizePath()` из Claude Code CLI (исходник: `claude-code-sourcecode/utils/sessionStoragePortable.ts:311`): все символы, кроме букв и цифр, заменяются на дефис. Это внешний контракт — любое отклонение от реального алгоритма Claude Code CLI приводит к тому, что бот ищет несуществующую папку сессий.

**Аргументы:**
- `project_dir` (str) — абсолютный путь к проекту. Например: `"/Users/ivan/Desktop/my project"`

**Возвращает:** закодированная строка. Например: `"-Users-ivan-Desktop-my-project"`

### `async def _read_session_file(file_path: str) -> SessionInfo | None`

Читает один JSONL-файл сессии и извлекает метаданные.

**Аргументы:**
- `file_path` (str) — абсолютный путь к `.jsonl` файлу сессии

**Возвращает:** объект `SessionInfo` с данными о сессии, или `None` если файл не удалось прочитать или он не содержит данных о сессии.

### `def _extract_first_user_message(lines: list[dict]) -> str`

Находит первое настоящее сообщение пользователя (не метаданные, не системную команду) и возвращает его текст.

**Аргументы:**
- `lines` (list[dict]) — список распарсенных JSON-строк из JSONL-файла

**Возвращает:** текст первого сообщения пользователя или пустую строку, если сообщение не найдено.

### `def _clean_preview(raw_text: str) -> str`

Очищает текст превью от служебных XML-тегов Claude Code и сжимает лишние пробелы.

**Аргументы:**
- `raw_text` (str) — необработанный текст сообщения пользователя, который может содержать XML-теги

**Возвращает:** очищенный текст без обязательной обрезки. Если после очистки текст пустой — пустая строка.

### `def _is_command_message(text: str) -> bool`

Проверяет, является ли текст сообщения системной/командной строкой Claude Code (содержит XML-теги из набора `COMMAND_XML_TAGS`).

**Аргументы:**
- `text` (str) — текст сообщения пользователя

**Возвращает:** `True` если текст содержит хотя бы один командный XML-тег, `False` иначе.

## Алгоритм работы

### get_recent_sessions

1. **Построить путь к папке сессий** — вызвать `_build_sessions_path(project_dir)` для получения пути вида `~/.claude/projects/-Users-ivan-Desktop-...`
2. **Проверить существование папки** — если папка не существует, залогировать предупреждение (уровень warning) и вернуть пустой список. Это штатная ситуация при первом запуске, когда ещё не было сессий
3. **Получить список JSONL-файлов** — прочитать содержимое папки через `asyncio.to_thread(os.listdir, ...)`, отфильтровать только файлы с расширением `.jsonl`. Подкаталоги и файл `memory` игнорировать
4. **Отсортировать по времени модификации** — для каждого `.jsonl` файла получить `os.path.getmtime()` через `asyncio.to_thread`, отсортировать по убыванию (новые первые)
5. **Ограничить количество** — взять первые 15 файлов из отсортированного списка (константа `MAX_RECENT_SESSIONS`)
6. **Прочитать каждый файл** — для каждого из 15 файлов вызвать `_read_session_file()`. Файлы читаются последовательно (не параллельно), чтобы не создавать нагрузку на диск при частых вызовах от watcher
7. **Отфильтровать ошибки** — убрать из списка результаты, которые вернули `None` (повреждённые файлы)
8. **Вернуть список** — список объектов `SessionInfo`

### get_session_messages

1. **Построить путь к файлу** — вызвать `_build_sessions_path(project_dir)` и добавить `{session_id}.jsonl`
2. **Проверить существование файла** — если файл не существует, залогировать предупреждение и вернуть пустой список
3. **Прочитать файл** — открыть файл и прочитать все строки через `asyncio.to_thread`. Каждую строку распарсить как JSON
4. **Обработать ошибки парсинга** — если строка не парсится как JSON, пропустить её и залогировать предупреждение. Не прерывать чтение остальных строк
5. **Вернуть список** — список словарей (распарсенные JSON-строки)

### _build_sessions_path

1. **Закодировать путь проекта** — вызвать `_encode_project_path(project_dir)` для преобразования пути в формат Claude Code
2. **Построить полный путь** — соединить домашнюю директорию (`~`), `.claude/projects/` и закодированное имя проекта
3. **Вернуть путь** — абсолютный путь к папке сессий

### _encode_project_path

1. **Применить регулярное выражение** — применить регулярное выражение `[^a-zA-Z0-9]` к пути и заменить все совпадения на `-`. Регулярка скомпилирована в константу модуля `SANITIZE_PATH_PATTERN` с комментарием-ссылкой на источник в Claude Code CLI.
2. **Вернуть результат** — закодированная строка. Путь `/Users/ivan/Desktop/my project` превращается в `-Users-ivan-Desktop-my-project`

### _read_session_file

1. **Прочитать первые строки файла** — открыть JSONL-файл и прочитать первые `MAX_LINES_FOR_PREVIEW` (50) строк через `asyncio.to_thread`. Не нужно читать весь файл — достаточно первых строк для извлечения метаданных и поиска первого пользовательского сообщения. Каждую строку распарсить как JSON, невалидные строки пропустить с логированием
2. **Извлечь session_id** — из первой распарсенной строки файла взять поле `sessionId` (через `first_line.get("sessionId")`). Если поле отсутствует — взять имя файла (без расширения `.jsonl`) как session_id. Claude Code использует UUID сессии в имени файла
3. **Извлечь время создания** — из первой распарсенной строки файла взять поле `timestamp` (через `first_line.get("timestamp")`). Если отсутствует — залогировать предупреждение и вернуть `None` (невозможно определить время создания, файл повреждён)
4. **Найти первое сообщение пользователя** — передать список распарсенных строк в `_extract_first_user_message()`
5. **Очистить превью** — передать найденный текст в `_clean_preview()`
6. **Собрать SessionInfo** — создать объект `SessionInfo(session_id=..., created_at=..., preview=...)` и вернуть

### _extract_first_user_message

1. **Перебрать строки** — идти по списку распарсенных JSON-строк
2. **Фильтр по типу** — искать строки, где `type == "user"` и `isMeta` отсутствует или равно `False`
3. **Извлечь текст** — из поля `message.content` извлечь текст. Content может быть:
   - Строкой — взять как есть
   - Списком — найти элемент с `type == "text"` и взять его поле `text`
4. **Пропустить системные сообщения** — вызвать `_is_command_message(text)`. Если вернула `True` — пропустить строку и искать дальше. Проверка основана на наличии XML-тегов из набора `COMMAND_XML_TAGS` (например, `<command-name>`, `<local-command-stdout>`, `<local-command-caveat>`)
5. **Пропустить пустые сообщения** — если текст после вызова `strip()` пуст или короче 2 символов, пропустить
6. **Вернуть первый подходящий** — текст первого сообщения, которое прошло все фильтры. Если ничего не нашлось — пустая строка

### _is_command_message

1. **Проверить наличие командных тегов** — для каждого тега из `COMMAND_XML_TAGS` проверить, содержится ли `<{tag_name}` в тексте (проверка через `in`, не через регулярное выражение — для скорости)
2. **Вернуть результат** — `True` если хотя бы один командный тег найден, `False` иначе

### _clean_preview

1. **Удалить XML-теги** — с помощью регулярного выражения `<[^>]+>` убрать все XML-теги из текста. Это удалит служебные теги Claude Code вроде `<command-name>`, `<task-notification>`, `<tool_uses>` и другие
2. **Убрать лишние пробелы** — заменить множественные пробелы и переносы строк на один пробел, обрезать пробелы по краям
3. **Вернуть результат** — очищенный текст. Константа `PREVIEW_MAX_LENGTH` сейчас равна `None`, поэтому список сессий не добавляет `"..."`

## Зависимости

- **config** — `WORKING_DIR` — абсолютный путь к рабочей директории проекта. Используется как аргумент `project_dir` при вызове из потребителей (session_manager, session_watcher, bot передают `config.WORKING_DIR` в функции session_reader)
- **os** (стандартная библиотека) — `os.listdir()`, `os.path.getmtime()`, `os.path.join()`, `os.path.exists()`, `os.path.isdir()`, `os.path.expanduser()` — работа с файловой системой
- **json** (стандартная библиотека) — `json.loads()` — парсинг JSON-строк из JSONL-файлов
- **re** (стандартная библиотека) — `re.sub()` — удаление XML-тегов из превью с помощью регулярных выражений
- **asyncio** (стандартная библиотека) — `asyncio.to_thread()` — вынос блокирующих операций (чтение файлов, обращение к файловой системе) в отдельный поток
- **logging** (стандартная библиотека) — логирование предупреждений и ошибок
- **dataclasses** (стандартная библиотека) — `@dataclass` — для определения класса `SessionInfo`

Зависимость от модулей проекта — только config (слой 0). session_reader сам не импортирует config напрямую, но потребители передают `config.WORKING_DIR` через аргумент `project_dir`.

**Внешние контракты:**
- Алгоритм формирования имени папки сессий — `sanitizePath()` из Claude Code CLI (`claude-code-sourcecode/utils/sessionStoragePortable.ts:311`). Любое изменение этой функции в Claude Code требует обновления `_encode_project_path` и регрессионных тестов.

## Обработка ошибок

- **Папка сессий не существует** — `_build_sessions_path` вернёт путь к несуществующей папке. `get_recent_sessions` проверяет существование папки и при её отсутствии логирует предупреждение (уровень warning): `"Папка сессий не найдена: {path}"` и возвращает пустой список. Это штатная ситуация при первом запуске
- **Папка пуста (нет JSONL-файлов)** — `get_recent_sessions` вернёт пустой список без ошибки. Залогирует на уровне info: `"Файлы сессий не найдены в {path}"`
- **JSONL-файл повреждён (невалидный JSON)** — `_read_session_file` ловит `json.JSONDecodeError`, логирует предупреждение: `"Ошибка чтения сессии {file_path}: невалидный JSON на строке {line_number}"`, возвращает `None`. Остальные файлы продолжают обрабатываться
- **JSONL-файл не содержит session_id** — `_read_session_file` использует имя файла (без `.jsonl`) как session_id. Это UUID, который Claude Code использует в имени файла
- **JSONL-файл не содержит timestamp** — `_read_session_file` возвращает `None` (невозможно определить время создания). Логирует предупреждение
- **Первое пользовательское сообщение не найдено** — `_extract_first_user_message` возвращает пустую строку. В SessionInfo поле `preview` будет пустым — потребители обработают это (покажут только номер сессии без превью)
- **Ошибка доступа к файлу (PermissionError)** — `_read_session_file` ловит исключение, логирует ошибку (уровень error): `"Нет доступа к файлу сессии: {file_path}"`, возвращает `None`
- **Файл сессии не найден (get_session_messages)** — логирует предупреждение и возвращает пустой список. session_watcher обработает это корректно
- **Ошибка при чтении os.listdir или os.path.getmtime** — `get_recent_sessions` ловит `OSError`, логирует ошибку и возвращает пустой список

## Константы

- `CLAUDE_PROJECTS_DIR = ".claude/projects"` — относительный путь (от домашней директории) к папке, где Claude Code хранит файлы проектов. Это стандартное расположение, заданное Claude Code
- `MAX_RECENT_SESSIONS = 15` — максимальное количество сессий, возвращаемых из `get_recent_sessions()`. Значение из BRD CJM-05: «бот берёт 15 самых свежих сессий»
- `PREVIEW_MAX_LENGTH = None` — лимит длины превью отключён, чтобы `/sessions` показывал полный очищенный заголовок без `"..."`
- `MAX_LINES_FOR_PREVIEW = 50` — максимальное количество строк JSONL-файла, которые читаются при поиске первого пользовательского сообщения. Ограничение нужно, чтобы не читать огромные файлы целиком при поиске превью
- `COMMAND_XML_TAGS = {"command-name", "command-message", "command-args", "local-command-stdout", "local-command-caveat"}` — набор XML-тегов, которые обозначают системные/командные сообщения Claude Code. Сообщения, содержащие эти теги, пропускаются при поиске первого пользовательского сообщения
- `XML_TAG_PATTERN = re.compile(r"<[^>]+>")` — скомпилированное регулярное выражение для удаления XML-тегов из превью. Компилируется один раз при загрузке модуля для эффективности
- `WHITESPACE_PATTERN = re.compile(r"\s+")` — скомпилированное регулярное выражение для замены множественных пробелов и переносов строк на один пробел

### Граничные случаи
