# Спецификация модуля: config

Дата: 30-03-2026
Слой: 0 (нет зависимостей от других модулей проекта)
Файл: `src/claude_manager/config.py`

## Назначение

Загружает настройки приложения из файла `.env` и проверяет, что все обязательные параметры заполнены. Предоставляет остальным модулям доступ к настройкам через типизированные константы уровня модуля.

## Обслуживаемые сценарии

- **CJM-01: Первый запуск и настройка** — при старте бота config загружает `.env`, проверяет обязательные поля (TELEGRAM_BOT_TOKEN, ALLOWED_USER_IDS), и если чего-то не хватает — останавливает программу с понятным сообщением об ошибке
- **CJM-09: Защита от двойного запуска** — предоставляет путь к рабочей директории (CLAUDE_WORKING_DIR), который используется модулем main для создания файла-замка bot.pid

## Публичный API

Модуль config экспортирует функцию загрузки и константы уровня модуля. Все потребители импортируют нужные значения напрямую: `from claude_manager.config import BOT_TOKEN, ALLOWED_USER_IDS`.

### `load_config() -> None`

Загружает настройки из `.env` файла и записывает их в константы уровня модуля. Проверяет, что обязательные параметры заполнены. Если обязательный параметр отсутствует или пуст — выбрасывает исключение.

**Аргументы:** нет

**Возвращает:** ничего (None). Результат работы — заполненные константы модуля.

**Исключения:**
- `ConfigError` — если обязательный параметр (TELEGRAM_BOT_TOKEN или ALLOWED_USER_IDS) отсутствует или пуст

### `class ConfigError(Exception)`

Исключение для ошибок конфигурации. Содержит понятное сообщение о том, какой именно параметр отсутствует.

### Константы модуля (заполняются после вызова `load_config()`)

- `BOT_TOKEN: str` — токен Telegram-бота (значение TELEGRAM_BOT_TOKEN из `.env`)
- `ALLOWED_USER_IDS: set[int]` — множество Telegram-ID пользователей, которым разрешён доступ к боту (значение ALLOWED_USER_IDS из `.env`, разобранное из строки через запятую)
- `WORKING_DIR: str` — абсолютный путь к рабочей директории проекта (значение CLAUDE_WORKING_DIR из `.env`, или текущая директория если не указано)
- `OPERATIONAL_SESSION_LOOKBACK_DAYS: int = 4` — окно последних дней для горячих operational-путей: watcher reset/poll и pending-сбор при `/pN`. Для Codex это окно обслуживается in-memory operational index, поэтому повторные обращения не перечитывают всю глобальную историю `~/.codex/sessions`.

## Внутренние функции

### `_parse_allowed_user_ids(raw_value: str) -> set[int]`

Разбирает строку с Telegram-ID пользователей, разделённых запятыми, в множество целых чисел.

**Аргументы:**
- `raw_value` (str) — строка вида `"123456789,987654321"` или `"123456789"` (один ID)

**Возвращает:** множество целых чисел — `set[int]`

**Исключения:**
- `ConfigError` — если строка содержит нечисловое значение (например, `"abc,123"`)
- `ConfigError` — если после разбора множество пустое (все элементы были пробелами или пустыми строками)

### `_resolve_working_dir(raw_value: str | None) -> str`

Определяет абсолютный путь к рабочей директории. Если значение задано — проверяет, что директория существует. Если не задано — возвращает текущую директорию.

**Аргументы:**
- `raw_value` (str | None) — значение CLAUDE_WORKING_DIR из `.env`, или None если не задано

**Возвращает:** абсолютный путь к рабочей директории — `str`

**Исключения:**
- `ConfigError` — если указанная директория не существует

## Алгоритм работы

### load_config

1. **Загрузить `.env` файл** — вызвать `dotenv.load_dotenv(override=True)`. Параметр `override=True` нужен, чтобы значения из `.env` перезаписывали системные переменные окружения (требование BRD CJM-01: «Настройки из .env перезаписывают системные»)
2. **Прочитать TELEGRAM_BOT_TOKEN** — получить значение из `os.environ`. Если отсутствует или пустая строка — выбросить `ConfigError("TELEGRAM_BOT_TOKEN не задан. Укажите токен бота в файле .env")`
3. **Прочитать ALLOWED_USER_IDS** — получить значение из `os.environ`. Если отсутствует или пустая строка — выбросить `ConfigError("ALLOWED_USER_IDS не задан. Укажите Telegram-ID разрешённых пользователей в файле .env")`. Если задано — передать в `_parse_allowed_user_ids()` для разбора
4. **Прочитать CLAUDE_WORKING_DIR** — получить значение из `os.environ` (может отсутствовать — это необязательный параметр). Передать в `_resolve_working_dir()` для определения абсолютного пути
5. **Записать значения в константы модуля** — присвоить `BOT_TOKEN`, `ALLOWED_USER_IDS`, `WORKING_DIR` глобальным переменным модуля
6. **Залогировать успех** — записать в лог (уровень info): какая рабочая директория используется и сколько пользователей в белом списке. Токен бота НЕ логировать (секрет)

### _parse_allowed_user_ids

1. **Разделить строку по запятым** — `raw_value.split(",")`
2. **Убрать пробелы** — для каждого элемента вызвать `.strip()`
3. **Отфильтровать пустые строки** — пропустить элементы, которые после strip() стали пустыми (например, `"123,,456"` — между запятыми пусто)
4. **Преобразовать в числа** — для каждого непустого элемента вызвать `int()`. Если преобразование не удалось (ValueError) — выбросить `ConfigError(f"ALLOWED_USER_IDS содержит нечисловое значение: '{element}'")`
5. **Проверить, что множество не пустое** — если после всех преобразований множество пустое, выбросить `ConfigError("ALLOWED_USER_IDS не содержит ни одного корректного ID")`
6. **Вернуть множество** — `set[int]`

### _resolve_working_dir

1. **Проверить, задано ли значение** — если `raw_value` равно None или пустой строке, вернуть `os.getcwd()` (текущая директория)
2. **Преобразовать в абсолютный путь** — `os.path.abspath(raw_value)`. Это обработает и относительные пути (например, `"./my-project"`)
3. **Проверить существование директории** — `os.path.isdir(resolved_path)`. Если директория не существует — выбросить `ConfigError(f"CLAUDE_WORKING_DIR указывает на несуществующую директорию: '{resolved_path}'")`
4. **Вернуть абсолютный путь** — строка `str`

## Зависимости

Модуль config зависит только от стандартной библиотеки Python и одного стороннего пакета:

- **os** (стандартная библиотека) — `os.environ`, `os.getcwd()`, `os.path.abspath()`, `os.path.isdir()` — чтение переменных окружения и работа с путями
- **logging** (стандартная библиотека) — логирование результатов загрузки
- **dotenv** (сторонний пакет python-dotenv 1.1.0) — `load_dotenv(override=True)` — загрузка переменных из `.env` файла

Зависимостей от других модулей проекта нет (слой 0).

## Обработка ошибок

- **Нет файла `.env`** — `dotenv.load_dotenv()` не выбрасывает ошибку, если файл отсутствует. Переменные останутся незаданными. Далее сработает проверка обязательных полей, и `ConfigError` укажет, какой параметр отсутствует
- **TELEGRAM_BOT_TOKEN отсутствует или пуст** — `ConfigError("TELEGRAM_BOT_TOKEN не задан. Укажите токен бота в файле .env")`. Программа должна завершиться (обрабатывается в main.py)
- **ALLOWED_USER_IDS отсутствует или пуст** — `ConfigError("ALLOWED_USER_IDS не задан. Укажите Telegram-ID разрешённых пользователей в файле .env")`. Программа должна завершиться
- **ALLOWED_USER_IDS содержит нечисловое значение** — `ConfigError("ALLOWED_USER_IDS содержит нечисловое значение: 'abc'")`. Программа должна завершиться
- **ALLOWED_USER_IDS содержит только пробелы и запятые** — `ConfigError("ALLOWED_USER_IDS не содержит ни одного корректного ID")`. Программа должна завершиться
- **CLAUDE_WORKING_DIR указывает на несуществующую директорию** — `ConfigError("CLAUDE_WORKING_DIR указывает на несуществующую директорию: '/path/to/nowhere'")`. Программа должна завершиться
- **CLAUDE_WORKING_DIR не задан** — штатная ситуация, используется текущая директория. Логируется на уровне info
- **load_config() вызван повторно** — допустимо, значения перезаписываются. Это может понадобиться в тестах

## Константы

- `_ENV_BOT_TOKEN = "TELEGRAM_BOT_TOKEN"` — имя переменной окружения для токена бота. Вынесено в константу, чтобы не дублировать строку в коде и в сообщениях об ошибках
- `_ENV_ALLOWED_IDS = "ALLOWED_USER_IDS"` — имя переменной окружения для списка разрешённых пользователей
- `_ENV_WORKING_DIR = "CLAUDE_WORKING_DIR"` — имя переменной окружения для рабочей директории

## Тест-план

### Юнит-тесты

- **test_load_config_all_values_set** — проверяет, что при наличии всех переменных окружения константы модуля заполняются корректно
  - Вход: `TELEGRAM_BOT_TOKEN="7654321:AAH_test_token_value"`, `ALLOWED_USER_IDS="123456789"`, `CLAUDE_WORKING_DIR="/tmp"`
  - Ожидаемый результат: `BOT_TOKEN == "7654321:AAH_test_token_value"`, `ALLOWED_USER_IDS == {123456789}`, `WORKING_DIR == "/tmp"`
  - Тип: unit

- **test_load_config_multiple_user_ids** — проверяет разбор нескольких ID через запятую
  - Вход: `ALLOWED_USER_IDS="123456789,987654321,555000111"`
  - Ожидаемый результат: `ALLOWED_USER_IDS == {123456789, 987654321, 555000111}`
  - Тип: unit

- **test_load_config_working_dir_defaults_to_cwd** — проверяет, что при отсутствии CLAUDE_WORKING_DIR используется текущая директория
  - Вход: `TELEGRAM_BOT_TOKEN="7654321:AAH_test_token_value"`, `ALLOWED_USER_IDS="123456789"`, CLAUDE_WORKING_DIR не задан
  - Ожидаемый результат: `WORKING_DIR == os.getcwd()`
  - Тип: unit

- **test_parse_allowed_user_ids_single_id** — проверяет разбор одного ID
  - Вход: `"123456789"`
  - Ожидаемый результат: `{123456789}`
  - Тип: unit

- **test_parse_allowed_user_ids_multiple_ids** — проверяет разбор нескольких ID
  - Вход: `"123456789, 987654321, 555000111"`
  - Ожидаемый результат: `{123456789, 987654321, 555000111}`
  - Тип: unit

- **test_resolve_working_dir_absolute_path** — проверяет, что абсолютный путь к существующей директории принимается как есть
  - Вход: `"/tmp"`
  - Ожидаемый результат: `"/tmp"`
  - Тип: unit

- **test_resolve_working_dir_none_returns_cwd** — проверяет, что None возвращает текущую директорию
  - Вход: `None`
  - Ожидаемый результат: `os.getcwd()`
  - Тип: unit

### Граничные случаи

- **test_user_ids_with_extra_spaces** — проверяет, что пробелы вокруг ID корректно убираются
  - Вход: `"  123456789 , 987654321  "`
  - Ожидаемый результат: `{123456789, 987654321}`
  - Тип: edge case

- **test_user_ids_with_trailing_comma** — проверяет, что запятая в конце строки не вызывает ошибку
  - Вход: `"123456789,987654321,"`
  - Ожидаемый результат: `{123456789, 987654321}`
  - Тип: edge case

- **test_user_ids_with_consecutive_commas** — проверяет, что несколько запятых подряд не вызывают ошибку
  - Вход: `"123456789,,987654321"`
  - Ожидаемый результат: `{123456789, 987654321}`
  - Тип: edge case

- **test_user_ids_duplicate_values** — проверяет, что дубликаты ID отбрасываются (множество)
  - Вход: `"123456789,123456789,987654321"`
  - Ожидаемый результат: `{123456789, 987654321}`
  - Тип: edge case

- **test_working_dir_relative_path** — проверяет, что относительный путь преобразуется в абсолютный
  - Вход: `"."` (текущая директория, всегда существует)
  - Ожидаемый результат: `os.path.abspath(".")` (абсолютный путь)
  - Тип: edge case

- **test_working_dir_empty_string** — проверяет, что пустая строка обрабатывается как «не задано»
  - Вход: `""`
  - Ожидаемый результат: `os.getcwd()`
  - Тип: edge case

- **test_load_config_env_overrides_system** — проверяет, что .env перезаписывает системные переменные (параметр override=True)
  - Вход: системная переменная `TELEGRAM_BOT_TOKEN="old_token"`, в `.env` записано `TELEGRAM_BOT_TOKEN=new_token`
  - Ожидаемый результат: `BOT_TOKEN == "new_token"`
  - Тип: edge case

- **test_load_config_can_be_called_twice** — проверяет, что повторный вызов load_config() перезаписывает значения без ошибок
  - Вход: первый вызов с `ALLOWED_USER_IDS="111"`, второй с `ALLOWED_USER_IDS="222"`
  - Ожидаемый результат: после второго вызова `ALLOWED_USER_IDS == {222}`
  - Тип: edge case

  - Ожидаемый результат: `ConfigError` с текстом, содержащим `"ALLOWED_USER_IDS не задан"`
  - Тип: error

- **test_empty_allowed_user_ids** — проверяет ошибку при пустом списке пользователей

- **test_nonexistent_working_dir** — проверяет ошибку при указании несуществующей рабочей директории

  - Ожидаемый результат: `ConfigError` с текстом, содержащим `"несуществующую директорию"` и `"/path/that/definitely/does/not/exist"`

- **test_working_dir_is_file_not_directory** — проверяет ошибку, когда путь указывает на файл, а не на директорию
  - Вход: `CLAUDE_WORKING_DIR="/tmp/some_test_file.txt"` (файл создаётся в setUp теста)
  - Ожидаемый результат: `ConfigError` с текстом, содержащим `"несуществующую директорию"`
  - Тип: error
