### Idempotency and drift tests

1. `--dry-run` не меняет дерево: checksum до и после совпадает.

5. Локальный drift в Codex mirror перезаписывается из Claude-source при следующем write.

### `_shared-validators` gate tests

1. Mirror или manifest без `_shared-validators` вызывает failure.
2. Pipeline skill с copied local enforcement вместо shared validator вызывает failure.
3. Critical check, пониженный до warning/non-critical, вызывает failure.
4. Generated Codex `structural-validator.sh`, который не может source mirrored shared validators, вызывает failure.
5. Source `check-enforcement-consistency.sh` и mirrored equivalent возвращают совпадающие critical results после canonical path transform.

## Допущения

- Claude-источник остаётся главным местом редактирования скиллов.
- Codex-дубликаты нужны как полноценные runtime-файлы для чтения Codex, но не как независимая ветка развития.
- Отдельный `.agents/skill-extensions/` не создаётся на первом этапе.
- Реализация выполняется отдельным коммитом миграции, потому что дифф будет крупным.
- Adapter contract может остановить Codex-выполнение, если нужный Claude-only механизм недоступен и fallback не описан; это лучше, чем silently changing pipeline semantics.
