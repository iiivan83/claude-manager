# Recent Sessions v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Реализовать persistent-таблицу `recent_sessions`, чтобы `/sessions` и `/all` брали быстрый список последних сессий без повторного полного обхода истории Codex.

**Architecture:** Новые модули `recent_sessions_store` и `recent_sessions_refresh` разделяют хранение SQLite и bounded refresh. `telegram_session_handlers.py` и `all_projects_monitor.py` остаются тонкими интеграционными слоями: они запрашивают готовые rows, регистрируют дневные номера и доставляют сообщения, но не содержат schema, pruning или discovery-алгоритмы. `/all` сохраняет snapshot кандидатов при включении и в poll-loop проверяет только известные файлы по mtime/cursor-first пути.

**Tech Stack:** Python 3.13, asyncio, sqlite3 из stdlib через `asyncio.to_thread`, pytest, pytest-asyncio, существующие backend adapters, `daily_session_registry`, `project_manager`, `session_watcher`, `unread_buffer`.

---

**Парная спецификация:** `dev/docs/specs/31.05_19.43-recent-sessions-v1-spec.md`

**Короткий контекст для исполнителя.** Claude Manager - локальный Telegram-бот, который показывает Ивану диалоги CLI-агентов. Проблема именно в пользовательском hot path: команда `/sessions`, включение `/all` и 2-секундный poll `/all` не должны снова обходить всю историю Codex, потому что Codex хранит rollout-файлы в общей папке `~/.codex/sessions/`, а проект узнаётся только после чтения metadata из JSONL.

**Термины, которые важны в этой работе:**
- **Hot path** - путь, который блокирует ответ Ивану или часто повторяется в фоне. Здесь это `/sessions`, `/all enable` и `/all poll`.
- **Bounded refresh** - ограниченное обновление индекса: по lookback-окну, известному session id, cap или timeout. Это разрешённый способ лечить пустую или устаревшую таблицу.
- **Candidate snapshot** - список сессий, который `/all` зафиксировал при включении режима. Poll-loop работает по нему и не делает discovery на каждом цикле.
- **Forbidden scan assertion** - тест со spy/counter, который падает, если код снова вызывает полный historical scan.

**Ограничение по размеру файлов на старте.** Перед изменением code modules обязательно пересчитать строки и top-level публичные функции. Текущий baseline на момент плана:

from __future__ import annotations

import asyncio
import os
import sqlite3
from dataclasses import dataclass
from datetime import datetime, timezone
from pathlib import Path

from claude_manager.coding_agent_backend import BackendName


RECENT_SESSIONS_FILENAME = "recent_sessions.sqlite3"
DEFAULT_DB_PATH = Path.home() / ".local" / "state" / "claude-manager" / RECENT_SESSIONS_FILENAME
SCHEMA_VERSION = "1"
PROJECT_STORAGE_LIMIT = 30
CURSOR_SCOPE_PROJECT = "project"
CURSOR_SCOPE_ALL = "all"


@dataclass(frozen=True)
class RecentSessionHeader:
    """One persisted session-list candidate."""

    project_path: str
    backend: BackendName
    session_id: str
    file_path: str
    last_modified_at: float
    preview: str = ""
    raw_record_count: int | None = None
    cursor_record_count: int | None = None
    file_missing: bool = False
    refresh_status: str = "ok"
    stale_reason: str = ""


@dataclass(frozen=True)
class RecentSessionCursorState:
    """Independent cursor state that is not pruned with header rows."""

    project_path: str
    backend: BackendName
    session_id: str
    file_path: str
    cursor_scope: str
    raw_record_count: int | None
    last_delivered_idx: int | None
    last_modified_at: float | None


class RecentSessionsStoreError(Exception):
    """Raised for degraded persistent-store operations."""

- [ ] **Step 3: Implement async facade through `asyncio.to_thread`**

The public class must expose this API:
```python
class RecentSessionsStore:
    """Async facade around SQLite recent session storage."""

    def __init__(self, db_path: Path = DEFAULT_DB_PATH) -> None:
        self.db_path = db_path
        self._write_lock = asyncio.Lock()

    async def initialize(self) -> None:
        """Initialize schema idempotently."""

    async def upsert_headers(self, headers: list[RecentSessionHeader]) -> None:
        """Insert or update headers and prune each touched project to 30 rows."""

    async def query_project(self, project_path: str, limit: int = PROJECT_STORAGE_LIMIT) -> list[RecentSessionHeader]:
        """Return active rows for one project by stable recent order."""

    async def query_global(self, limit: int) -> list[RecentSessionHeader]:
        """Return active rows across projects by stable recent order."""

    async def mark_missing(self, project_path: str, backend: BackendName, session_id: str) -> None:
        """Hide one header row from active queries while preserving cursor state."""

    async def upsert_cursor_state(self, state: RecentSessionCursorState) -> None:
        """Persist independent cursor state."""

    async def get_cursor_state(
        self,
        project_path: str,
        backend: BackendName,
        session_id: str,
        cursor_scope: str,
    ) -> RecentSessionCursorState | None:
        """Return independent cursor state for one scope."""
```

Implementation requirements:
- All sqlite work runs inside private blocking helpers called with `asyncio.to_thread`.
- `initialize()` creates parent directory with `mkdir(parents=True, exist_ok=True)`.
- Every connection sets `PRAGMA busy_timeout = 1000`.
- `upsert_headers()` uses one transaction under `_write_lock`, writes `created_at` only on insert, updates `updated_at` and `last_refreshed_at`, then prunes per touched `project_path`.
- Pruning deletes only from `recent_sessions`, never from `session_cursor_state`.
- Query methods filter `file_missing = 0` and sort by `last_modified_at DESC, updated_at DESC, backend ASC, session_id ASC`.
- Convert sqlite errors to `RecentSessionsStoreError` in async public methods.
