# ADR: проверять Python-процесс в restart post-flight

- **Дата:** 14-05-2026
- **Статус:** принято
- **Пайплайн:** session-change-documenter
- **Агент:** session-change-documenter (standalone)
- **Скилл:** session-change-documenter
- **Сущность:** `restart-claude-manager.sh`

## Контекст

`restart-claude-manager.sh` делал preflight, затем `launchctl kickstart -k`, затем post-flight через `launchctl list`. После restart macOS мог показывать строку вида `32325 -15 com.ivan.claude-manager`: первый столбец означал живой процесс launchd job, а второй столбец оставался stale-кодом завершения старого процесса.

Старый подход считал `-15` провалом даже когда бот уже поднялся. Первый фикс по PID тоже был недостаточен: PID в `launchctl list` принадлежит shell-обёртке `start-claude-manager.sh`, а обёртка может быть жива во время retry-паузы, пока Python-бот ещё не запущен.

## Решение

Post-flight считается успешным только если выполнены оба условия:

- `launchctl list` показывает числовой PID wrapper-процесса;
- в таблице процессов есть дочерний Python-процесс с запуском `runpy._run_module_as_main("claude_manager")`.

`last exit code = -15` остаётся диагностическим контекстом, но не считается ошибкой, если реальный Python-бот уже работает.
