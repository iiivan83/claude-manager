# ADR: компактный двухстрочный бюджет заголовка в `/sessions`

- **Дата:** 01-06-2026
- **Статус:** принято
- **Пайплайн:** session-change-documenter
- **Агент:** session-change-documenter (standalone)
- **Скилл:** session-change-documenter
- **Сущность:** `/sessions`, Telegram list row, cached preview

## Контекст

Список `/sessions` нужен для быстрого выбора старого диалога в Telegram. Предыдущее решение убрало из строки полное название backend-а и стало показывать полный очищенный заголовок исходного запроса.

На практике длинные заголовки начали занимать слишком много места в одном Telegram-сообщении. Пользователь хотел видеть больше смысла, чем раньше, но при этом держать каждую строку списка в компактном двухстрочном бюджете, чтобы список оставался сканируемым на телефоне.

## Решение

Строка `/sessions` остаётся plain text с кликабельной командой `/N` и короткой backend-иконкой. Сам заголовок теперь ограничивается `SESSION_LIST_TITLE_MAX_LENGTH = 120` символами и суффиксом `...`.

Если в `recent_sessions` лежит старый cached preview, который уже был обрезан, handler сначала пробует восстановить полный первый пользовательский запрос из файла сессии. После восстановления всё равно применяется компактный бюджет строки. Так список не возвращается к старому короткому preview, но и не расползается на большие блоки текста.

## Связанные файлы

- **`src/claude_manager/telegram_session_handlers.py`** — добавляет `_compact_session_list_title()` и применяет лимит к каждой строке `/sessions`.
- **`tests/test_telegram_session_list_format.py`** — фиксирует, что длинные заголовки в итоговом Telegram-сообщении сжимаются до лимита.
- **`dev/docs/brd/brd-user-journeys.md`** — описывает для пользователя, что список показывает компактный очищенный заголовок.

## Заменяет

Частично уточняет ADR `dev/docs/adr/01.06_00.40-session-change-documenter-session-list-full-titles.md`: полный preview всё ещё может использоваться как источник для восстановления смысла заголовка, но UI-строка больше не обязана показывать его целиком.

## Примечания

Лимит количества строк списка не изменился: `/sessions` по-прежнему показывает максимум 15 свежих сессий.
