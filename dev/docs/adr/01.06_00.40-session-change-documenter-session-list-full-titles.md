# ADR: полный заголовок сессии в списке `/sessions`

- **Дата:** 01-06-2026
- **Статус:** принято
- **Пайплайн:** session-change-documenter
- **Агент:** session-change-documenter (standalone)
- **Скилл:** session-change-documenter
- **Сущность:** `/sessions`, `SessionFileInfo.preview`, cached `recent_sessions` preview

## Контекст

Команда `/sessions` показывает Ивану последние диалоги Telegram-бота Claude Manager. Это быстрый список для выбора старой сессии: пользователь смотрит номер, заголовок и нажимает `/N`.

До изменения строка списка включала полное имя backend-а (`⚡ Codex` или `🤖 Claude`) и обрезала заголовок исходного запроса до 120 символов с `...`. На практике это занимало место в узком Telegram-окне и скрывало самую полезную часть заголовка, особенно у длинных задач.

После появления persistent-индекса `recent_sessions` часть заголовков уже могла быть сохранена в старом обрезанном виде. Простое отключение лимита для новых refresh-ов не исправляло бы первый показ таких строк.

## Решение

В строках `/sessions` показывать только короткую иконку backend-а, а не полное название модели:

```text
/131 ⚡ Смержить готовую ветку recent sessions в main
```

Политика `preview` для списка сессий меняется с «очищенный текст до 120 символов» на «полный очищенный текст без обязательной обрезки». Для новых данных `PREVIEW_MAX_LENGTH` установлен в `None`.

Если в `recent_sessions` лежит старый заголовок, который уже заканчивается на `...`, обработчик `/sessions` пытается дочитать файл сессии через backend-reader и восстановить полный первый пользовательский запрос. Для Codex при этом пропускается служебный bootstrap-блок `# AGENTS.md instructions`, чтобы не вернуть технический текст вместо темы сессии.

## Связанные файлы

- **`src/claude_manager/telegram_session_handlers.py`** — форматирует строку `/sessions` без слова `Claude`/`Codex` и дочитывает полный preview для старого обрезанного кэша.
- **`src/claude_manager/session_request_preview.py`** — сохраняет возможность лимита, но по умолчанию не обрезает очищенный текст.
- **`src/claude_manager/session_reader.py`** — отключает legacy-лимит `PREVIEW_MAX_LENGTH`.
- **`src/claude_manager/claude_code_session_file_reader.py`** — отключает лимит preview для Claude JSONL.
- **`src/claude_manager/codex_session_file_reader.py`** — отключает лимит preview для Codex rollout JSONL.
- **`src/claude_manager/session_summary_generator.py`** — увеличивает подсказку генератору summary до 160 символов и не обрезает сгенерированную строку молча.
- **`tests/test_telegram_session_handlers.py`** — фиксирует новый формат строки без названия backend-а.
- **`tests/test_telegram_session_list_format.py`** — фиксирует восстановление полного заголовка из старого cached-preview с `...`.
- **`tests/test_session_request_preview.py`**, **`tests/test_session_reader.py`**, **`tests/test_session_summary_generator.py`** — фиксируют отсутствие молчаливой обрезки.
- **`dev/docs/brd/brd-user-journeys.md`** — обновляет пользовательский сценарий `/sessions`.
- **`dev/docs/specs/realised/*`** — синхронизирует реализованные технические контракты с новой политикой preview.

## Примечания

Это решение не меняет лимит количества строк в `/sessions`: список по-прежнему показывает максимум 15 свежих сессий. Меняется только содержимое одной строки и политика длины заголовка.
