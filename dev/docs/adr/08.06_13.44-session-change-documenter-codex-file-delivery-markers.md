# ADR: Codex использует существующие Telegram-маркеры доставки файлов

**Дата:** 08-06-2026
**Статус:** принято
**Пайплайн:** session-change-documenter
**Агент:** session-change-documenter (standalone)
**Скилл:** session-change-documenter
**Сущность:** Codex backend, Telegram-доставка файлов, маркеры `[SEND_FILE:...]` и `[SHOW_FILE:...]`

## Контекст

Telegram-бот уже умеет доставлять файлы из финального ответа агента. Для этого агент вставляет в текст ответа служебные маркеры: `[SEND_FILE:/absolute/path]` для отправки файла вложением и `[SHOW_FILE:/absolute/path]` для показа содержимого текстового файла в чате.

Для Claude этот договор уже был описан в проектных инструкциях и фактически работал. Codex запускался через `codex exec ... prompt_text` и не получал отдельного правила про эти маркеры. Поэтому при просьбах «пришли файл» или «покажи содержимое файла» Codex мог отвечать текстом, но не запускал существующий механизм Telegram-доставки.

Рассматривались два нежелательных направления:

- Добавить новый механизм доставки специально для Codex. Это дублировало бы уже работающий контур `file_sender.py`, `file_delivery.py` и `telegram_response_delivery.py`.
- Записать правило глобально в `~/.codex/AGENTS.md`. Тогда оно действовало бы во всех Codex-сессиях, даже вне Telegram-бота.

## Решение

Codex backend добавляет короткую bot-mode инструкцию прямо в prompt, который передаётся в `codex exec`.

Инструкция говорит Codex:

- если пользователь просит прислать, отправить или приложить файл — вставить `[SEND_FILE:/absolute/path]` с реальным абсолютным путём;
- если пользователь просит показать, прочитать или вывести содержимое файла — вставить `[SHOW_FILE:/absolute/path]` с реальным абсолютным путём;
- не объяснять маркер пользователю, потому что бот вырежет его из финального ответа и выполнит действие.

Один и тот же prompt-wrapper применяется для новой Codex-сессии и для `resume` существующей сессии. Пользовательский текст сохраняется в конце prompt без потери.

## Связанные файлы

- **`src/claude_manager/codex_backend.py`** — добавляет `BOT_MODE_FILE_DELIVERY_PROMPT_APPENDIX` и применяет его в `compose_subprocess_command_args`.
- **`tests/test_codex_backend_bot_mode_prompt.py`** — проверяет, что новая и resumed Codex-сессии получают правила про `[SEND_FILE:...]` и `[SHOW_FILE:...]`.
- **`tests/test_codex_backend.py`** — старые тесты argv обновлены так, чтобы проверять структуру команды отдельно от обёрнутого prompt.
- **`dev/docs/brd/brd-user-journeys.md`** — сценарий доставки файлов описан как общий сценарий агента, а не только Claude.
- **`CLAUDE.md`** — проектная инструкция обновлена с Claude-specific wording на backend-neutral wording.

## Примечания

Контур доставки файлов не менялся. Маркеры по-прежнему обрабатываются только в финальных ответах, а `file_delivery.py` остаётся владельцем отправки файлов и показа содержимого в Telegram.
