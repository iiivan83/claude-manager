# ADR: разбиение process_manager по ответственностям

**Дата:** 10-06-2026
**Статус:** принято
**Пайплайн:** session-change-documenter
**Агент:** session-change-documenter (standalone)
**Скилл:** session-change-documenter
**Сущность:** структура инфраструктурного слоя управления CLI-процессами

## Контекст

`process_manager.py` управляет жизненным циклом процессов Claude/Codex CLI: запускает subprocess, отправляет пользовательское сообщение, читает stream-json события, делает retry и обрабатывает `/stop`. До этой сессии файл уже был больше 1000 строк, то есть стал срочным кандидатом на разбиение: любая новая правка требовала всё больше локального контекста и повышала риск случайно сломать retry, `/stop` или backend-aware путь.

Первый предыдущий разрез уже вынес in-memory state в `process_state.py`, но основное поведение всё ещё оставалось в одном файле. После точечного фикса файл снова вырос, и было принято решение не продолжать молча раздувать его дальше.

## Решение

Разбить `process_manager.py` на небольшие модули по ответственности, но сохранить `process_manager.py` как compatibility facade. Внешний код и старые тесты продолжают импортировать прежние имена из `claude_manager.process_manager`, а реальная логика теперь живёт в профильных файлах:

- `process_types.py` — общие типы, dataclass-результаты и callback-типы.
- `process_events.py` — чтение stream-json событий, progress/result и обновление session_id.
- `process_stop.py` — `/stop`, стратегия остановки процессов и `stop_all_processes`.
- `process_lifecycle.py` — создание temp session_id, запуск subprocess и restart.
- `process_retry.py` — retry-loop, ожидание с проверкой `/stop` и классификация permanent errors.
- `process_send.py` — `send_message` dispatcher и legacy Claude send path.
- `process_backend_send.py` — backend-aware Claude/Codex send path.

Ключевой принцип: это структурный рефакторинг без изменения пользовательского поведения. Поэтому тесты обновлены только там, где они патчили приватные внутренние точки старого монолита.

## Связанные файлы

- **`src/claude_manager/process_manager.py`** — оставлен как compatibility facade для старых импортов.
- **`src/claude_manager/process_types.py`** — создан как общий слой типов process lifecycle.
- **`src/claude_manager/process_events.py`** — создан для чтения событий и обработки progress/result.
- **`src/claude_manager/process_stop.py`** — создан для логики `/stop` и остановки всех процессов.
- **`src/claude_manager/process_lifecycle.py`** — создан для запуска и перезапуска subprocess.
- **`src/claude_manager/process_retry.py`** — создан для retry-loop и permanent-error classification.
- **`src/claude_manager/process_send.py`** — создан для dispatcher отправки и legacy Claude path.
- **`src/claude_manager/process_backend_send.py`** — создан для backend-aware Claude/Codex path.
- **`tests/test_process_manager.py`** — обновлён под новые внутренние patch-точки.
- **`tests/test_stop_triggers_retry_blackbox.py`** — обновлён под вынесенный retry/stop flow.
- **`tests/test_stop_triggers_retry_whitebox.py`** — обновлён под вынесенный retry/stop flow.
- **`tests/integration/test_cwd_pinning_across_retries.py`** — обновлён под новую точку retry-loop.

## Примечания

Размер `process_manager.py` снизился с 1233 до 84 строк. Все новые `process_*.py` файлы остались меньше 300 строк, поэтому следующий рефакторинг можно делать точечно, не продолжая god-module.
