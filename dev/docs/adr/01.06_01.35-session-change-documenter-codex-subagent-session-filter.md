# ADR: фильтр субагентских Codex-сессий в `recent_sessions`

- **Дата:** 01-06-2026
- **Статус:** принято
- **Пайплайн:** session-change-documenter
- **Агент:** session-change-documenter (standalone)
- **Скилл:** session-change-documenter
- **Сущность:** Codex `session_meta`, `recent_sessions`, `/sessions`, `/all`

## Контекст

Команда `/sessions` показывает Ивану последние диалоги, к которым можно вернуться. После подключения Codex и агентных workflows туда начали попадать не только обычные пользовательские сессии, но и spawned-сессии субагентов: reviewer, worker и другие технические роли.

По тексту заголовка такие сессии часто выглядят узнаваемо: «Ты code quality reviewer...», «Ты рабочий агент...». Но текстовая эвристика хрупкая: пользователь теоретически может начать обычный диалог похожими словами, а формат промптов субагентов может измениться.

В локальных Codex rollout JSONL есть внутренний признак происхождения. В `session_meta.payload` обычная сессия имеет `thread_source: user`, а субагентская - `thread_source: subagent` и `source.subagent.thread_spawn` с parent thread и ролью.

## Решение

User-facing список сессий должен опираться на внутреннюю metadata Codex, а не на текст первого сообщения.

Сессия считается субагентской и не попадает в пользовательский справочник, если:
- `session_meta.payload.thread_source == "subagent"`;
- или `session_meta.payload.source` содержит ключ `subagent`.

Фильтр применяется в двух местах:
- при чтении Codex rollout-файлов для свежего listing и operational index;
- при чтении уже сохранённых строк `recent_sessions`, чтобы старые cached-субагенты помечались скрытыми и больше не возвращались в `/sessions` или `/all`.

Отдельный helper `codex_session_metadata.py` вынесен из listing/index модулей, чтобы не дублировать знание о Codex metadata и не раздувать уже крупный `codex_session_file_listing.py`.

## Связанные файлы

- **`src/claude_manager/codex_session_metadata.py`** — новый helper для чтения Codex `session_meta` и определения субагентских rollout-файлов.
- **`src/claude_manager/codex_session_file_listing.py`** — исключает субагентов из user-facing Codex listing.
- **`src/claude_manager/codex_session_index.py`** — исключает субагентов из operational index, который питает быстрые горячие пути.
- **`src/claude_manager/recent_sessions_refresh.py`** — скрывает уже закэшированные Codex-субагенты через `mark_missing`.
- **`tests/test_codex_session_file_listing.py`** — фиксирует, что `/sessions`-listing не возвращает Codex-субагентов.
- **`tests/test_codex_session_index.py`** — фиксирует тот же фильтр для operational index.
- **`tests/test_recent_sessions_refresh.py`** — фиксирует очистку старых cached rows из `recent_sessions`.
- **`dev/docs/brd/brd-user-journeys.md`** — обновляет пользовательские сценарии `/sessions` и `/all`.
- **`dev/docs/adr/project_architecture.md`** — фиксирует новый контракт user-facing индекса сессий.

## Примечания

Фильтр не удаляет исходные JSONL-файлы Codex. Он только убирает технические spawned-сессии из пользовательского справочника и мониторингового snapshot.
