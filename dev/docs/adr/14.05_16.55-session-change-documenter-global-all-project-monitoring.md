# ADR: отдельный монитор для глобального режима all

- **Дата:** 14-05-2026
- **Статус:** принято
- **Пайплайн:** session-change-documenter
- **Агент:** session-change-documenter (standalone)
- **Скилл:** session-change-documenter
- **Сущность:** глобальный мониторинг проектов в режиме `/all`

## Контекст

До изменения `/all` был описан как режим просмотра сессий текущего проекта. После добавления переключения проектов этого стало недостаточно: пользователь должен видеть ответы агента из всех доступных проектов и быстро входить в нужный проект и сессию.

Обычный `session_watcher` хранит состояние доставки активного проекта. Если расширить его на все проекты, просмотр в `/all` мог бы преждевременно помечать сообщения прочитанными и ломать pending-доставку при входе в исходный проект.

## Решение

Для глобального `/all` используется отдельный модуль `all_projects_monitor`.

Он сканирует все проекты из `PROJECTS_ROOT_DIR` и все настроенные CLI-бэкенды, хранит собственные cursor-счётчики по ключу `(project_path, session_id, backend)` и показывает сообщения с кликабельной командой вида `/3s12`.

Перед показом сообщения в `/all` монитор сохраняет snapshot в `unread_buffer`, но не продвигает состояние обычного `session_watcher`. Поэтому сообщение, увиденное в общем режиме, остаётся pending для исходного проекта и доставляется повторно после входа в этот проект.

Если выход из `/all` через `/pN` или `/3s12` завершается ошибкой переключения проекта, бот заново включает `all_projects_monitor` и не возобновляет `session_watcher` поверх глобального режима.

## Связанные файлы

- **`src/claude_manager/all_projects_monitor.py`** — реализует отдельный глобальный монитор, cursor-состояние и link registry
- **`src/claude_manager/bot.py`** — подключает `/all`, строку `/all all`, обработчик `/3s12`, блокировки ввода и восстановление all-mode после ошибки переключения
- **`src/claude_manager/project_manager.py`** — предоставляет `collect_pending_messages_for_project` для выхода из all-mode в уже активный проект
- **`src/claude_manager/unread_buffer.py`** — хранит snapshot, по которому сообщение из `/all` остаётся pending для исходного проекта
- **`tests/test_all_projects_monitor.py`** — покрывает отдельный монитор и семантику unread snapshot
- **`tests/test_bot.py`** — покрывает Telegram-поведение `/all`, `/projects`, `/3s12` и блокировки ввода
- **`tests/test_project_manager.py`** — покрывает публичную pending-обёртку для уже активного проекта
