# Индекс скиллов

Живой документ — обновляется при добавлении, удалении или переименовании скиллов.

Все скиллы лежат в `.claude/skills/`. Правила для скиллов — в `.claude/skills/AGENTS.md`. Эталонные скиллы копируются из `~/.claude/skill-templates/` — если правишь такой скилл, правь сразу эталон.

Всего в проекте 33 скилла.

## Эталонные пайплайны разработки скиллов

Универсальные пайплайны для работы над самими скиллами. Копируются из `~/.claude/skill-templates/` в каждый проект.

- **skill-creator** — создание новых скиллов с нуля, включая eval-и и тестирование
- **pipeline-designer** — проектирование спецификаций пайплайнов от идеи до готового `spec.md`
- **pipeline-implementer** — сборка рабочих скиллов по готовой спецификации
- **pipeline-explorer** — исследование задачи обработки данных через итеративный цикл проб и проверок
- **update-skill** — полное обновление скиллов с бэкапом и сравнительным тестированием старой и новой версии через CLI
- **fast-skill-updater** — быстрое обновление скиллов за один цикл, без сравнительного прогона
- **script-creator-pipeline** — сквозной пайплайн «от идеи до готового скрипта»: спека, код, тесты, починка багов
- **feature-pipeline** — полный цикл доработки существующего функционала: спека, тесты, реализация, ревью, коммит

## Анализ и отладка

Скиллы для поиска первопричин проблем и починки багов.

- **root-cause-analysis** — глубокий анализ и поиск первопричины проблемы, работает в любом проекте
- **apply-root-cause-fixes** — системное применение исправлений по готовому root-cause отчёту с верификацией по 5 критериям
- **universal-bug-fixer** — универсальный пайплайн исправления ошибок с двойной верификацией результата

## Работа с документами

Создание и обновление произвольных документов проекта.

- **create-doc** — создание нового документа на основе диалога или контекста беседы
- **update-docs** — обновление существующих документов новой информацией из переписки

## Разработка модулей Claude Manager

Пишем спеку — потом реализуем. Четыре скилла для двух типов артефактов (модуль и скрипт).

- **spec-module** — написание детального технического задания на один модуль проекта
- **implement-module** — реализация модуля по спецификации: код, юнит-тесты, цикл починки до зелёных тестов
- **make-script-spec** — написание технического задания на внутренний скрипт разработки (не продакшен)
- **dev-script-implement** — реализация внутренних скриптов по готовой спеке

## Тестирование

Юнит-тесты проекта пишутся вручную, а интеграция и E2E — через эти скиллы.

- **test-integration** — написание и прогон интеграционных тестов по 5 группам взаимодействий
- **test-e2e** — живое E2E тестирование через Telethon: реальный Telegram, реальный Claude, LLM-верификация
- **autofix-e2e** — автоматический цикл починки всех падающих E2E тестов через root-cause и откат при неудаче
- **create-user-test-scenarios** — генерация пошаговых сценариев ручного тестирования из BRD
- **run-user-testing** — прогон пользовательских тест-сценариев с живым ботом и LLM-верификацией

## Пайплайны проекта

Оркестраторы и подготовительные этапы главного инженерного пайплайна Claude Manager.

- **pipeline-run** — главный оркестратор пайплайна: координирует все 10 фаз от валидации BRD до готового продукта
- **validate-brd** — глубокий технический аудит BRD перед реализацией, сверяет документы с реальным кодом
- **brd-decompose** — декомпозиция BRD на модули с графом зависимостей и порядком сборки по слоям
- **project-setup** — настройка инфраструктуры проекта перед разработкой: зависимости, тесты, фикстуры, логи

## Документация и отчёты проекта

Генерация и обновление документации по итогам работы.

- **session-report** — генерация отчёта о проделанной работе в текущей сессии
- **update-session-report** — обновление существующего отчёта вместо создания нового файла (борется с фрагментацией)
- **update-project-docs** — массовое обновление всей документации проекта на основе анализа реального кода

## Генерация скриптов инфраструктуры

Автогенерация вспомогательных bash- и Python-скриптов для проекта.

- **create-phase-gate-checker** — генерация Python-скрипта `check-phase-gate.py` для проверки ворот качества пайплайна
- **create-test-runner** — генерация bash-скрипта `run-all-tests.sh` для единого запуска юнит- и интеграционных тестов

## Рефакторинг и ревью

Массовые правки и сквозное ревью кода.

- **rename-entity** — переименование сущностей (файлы, папки, скиллы) с автоматической правкой всех ссылок
- **review-code** — глубокое 4-проходное ревью всего кода проекта с автоисправлением найденных проблем
