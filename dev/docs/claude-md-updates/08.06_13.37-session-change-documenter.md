# CLAUDE.md update: доставка файлов из ответа агента

**Дата:** 08-06-2026
**Пайплайн:** session-change-documenter
**Агент:** session-change-documenter (standalone)
**Скилл:** session-change-documenter

## Затронутая секция

Обзор ключевых возможностей, дерево структуры проекта и описание транспортного слоя.

## Было

`CLAUDE.md` описывал доставку файлов как возможность из ответа Claude. Это отражало старую формулировку, хотя контур доставки в Telegram-боте уже работает через общие маркеры `[SEND_FILE:path]` и `[SHOW_FILE:path]`.

## Стало

Формулировка заменена на «ответ агента», чтобы правило относилось и к Claude, и к Codex. Описание `file_delivery.py` и транспортного слоя также обновлено с Claude-specific wording на backend-neutral wording.

## Причина

Codex backend теперь добавляет в prompt Telegram bot-mode инструкцию: если пользователь просит прислать файл, Codex должен вставить `[SEND_FILE:/absolute/path]`; если просит показать содержимое файла, должен вставить `[SHOW_FILE:/absolute/path]`. Бот вырезает маркер из финального ответа и выполняет доставку.

## Связанные файлы

- **`src/claude_manager/codex_backend.py`** — добавляет инструкцию Codex про маркеры доставки файлов.
- **`dev/docs/brd/brd-user-journeys.md`** — сценарий доставки файлов обновлён с Claude-specific wording на общий сценарий для агента.
