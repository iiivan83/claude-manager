В **Принципах эксплуатации** были четыре буллета:

1. «Принцип verify-before-and-after для рестарта сервисов» — описывал `launchctl kickstart -k`, PID обёртки в `launchctl list`, stale `last exit code = -15`, путь `~/.local/bin/start-claude-manager.sh`.
2. «Принцип изоляции venv от TCC-защищённых директорий» — описывал `com.apple.provenance`, `UF_HIDDEN` на `.pth`-файлах, APFS-восстановление UF_HIDDEN, расположение venv в `~/.venvs/claude-manager/` с симлинком.
3. «Принцип изоляции скрипта запуска от TCC-зоны» — описывал блокировку launchd на bash-скриптах в `~/Desktop`, расположение скрипта в `~/.local/bin/start-claude-manager.sh` как обязательное.
4. «Принцип запрета самоперезапуска из собственного дерева процессов» — описывал цепочку «бот → Claude CLI → bash → kickstart → убивает бот», exit 137.

В секции **Принцип обязательного editable install** упоминались `~/.venvs/claude-manager/` с симлинком, APFS provenance, `UF_HIDDEN`, ссылка на «Принцип изоляции venv».

Отдельный раздел **Принцип retry-обёртки для launchd-сервисов** описывал shell-обёртку `~/.local/bin/start-claude-manager.sh` с 3 retry × 10 секунд, причину crash в `getpath.py` под Python 3.13.

В **Командах разработки**:

```

launchctl unload ~/Library/LaunchAgents/com.ivan.claude-manager.plist  # остановить

# Логи автозапуска

```

## Стало

В секции **Технологии**:

```
- **ОС:** Linux (systemd user services, fcntl для файловой блокировки доступен)
```

В секции **Структура проекта** строки про `start-claude-manager.sh` и `.venv` симлинк удалены, описание `restart-claude-manager.sh` обновлено: `preflight + systemctl restart + post-flight`.

В **Принципах эксплуатации** остались только два буллета:

1. **Переписан** «Принцип verify-before-and-after для рестарта сервисов» — теперь описывает `systemctl --user restart`, post-flight через `systemctl --user is-active` + `pgrep -f claude_manager`, stale exit codes через `systemctl reset-failed claude-manager.service`. Инцидент-источник 20-04-2026 сохранён.
2. **Переписан** «Принцип запрета самоперезапуска из собственного дерева процессов» — «launchd-сервис» заменён на «сервис под supervision», цепочка «bash → kickstart» заменена на «bash → systemctl restart».

Принципы изоляции venv и скрипта от TCC-зоны удалены полностью.

**Принцип обязательного editable install** переписан: venv живёт в `/home/ivan/claude-sandbox/claude_manager/.venv/`, упоминания TCC и APFS убраны, добавлено «`systemctl --user restart` упадёт с "No such file or directory"» как симптом сломанного editable install.

Отдельный раздел **Принцип retry-обёртки для launchd-сервисов** удалён полностью. На Linux эту роль играет systemd через `Restart=always` + `RestartSec=10` — обёртка не нужна.

В **Командах разработки**:

```
# Управление сервисом systemd (бот стартует автоматически после enable)
systemctl --user enable claude-manager.service   # включить автозапуск
systemctl --user start claude-manager.service    # запустить
systemctl --user stop claude-manager.service     # остановить
systemctl --user disable claude-manager.service  # убрать из автозапуска
systemctl --user status claude-manager.service   # статус

# Рестарт бота (preflight + systemctl restart + post-flight)
./restart-claude-manager.sh

# Логи приложения (RotatingFileHandler)
tail -f ~/.local/state/claude-manager/claude-manager.log

# Логи сервиса (stdout/stderr под systemd)
journalctl --user -u claude-manager.service -f
```

В **Важных деталях для разработки**:

- Буллет про `EDEADLK` удалён (macOS-специфика APFS, на Linux не возникает).
- Файловая блокировка: «единый путь для **systemd**, watch_and_restart.sh и ручного запуска».
- Переключение проектов: «`PROJECTS_ROOT_DIR` (обязательная переменная, без default)».
- Контракт имени папки сессий: «`CLAUDE_REAL_BIN` или `~/.npm-global/bin/claude`» (универсальный путь).
- Контракты с внешними системами: упоминание `~/Desktop/claude-sandbox/claude-code-sourcecode/` удалено целиком (этого пути на Linux нет).

## Причина

Пользователь полностью перевёл проект с macOS на Linux. Старая инфраструктура (launchd, TCC-защищённые директории Desktop, `~/Library/Logs/`) физически не существует на Linux. CLAUDE.md описывал принципы эксплуатации, привязанные к macOS, и эти принципы становились шумом для будущей работы — Claude (читая CLAUDE.md перед задачей) пытался бы применять macOS-специфичные правила в Linux-окружении, что приводило бы к запутыванию.

Изменения в CLAUDE.md синхронизированы с изменениями в самом коде (`bot.py`, `main.py`, `config.py`) и shell-скриптах (`restart-claude-manager.sh`, `watch_and_restart.sh`) — всё это часть единой миграции, описанной в `dev/docs/specs/28.05_17.25-linux-bot-commands-cleanup-spec.md`.

Принципы, которые не были привязаны к ОС, сохранены: запрет самоперезапуска из собственного дерева процессов работает одинаково для launchd и для systemd (subprocess в cgroup сервиса будет убит вместе с сервисом, exit 137), его формулировка только обобщена.
