# CLAUDE.md update: global all-project monitoring context

- **Дата:** 14-05-2026

- **Скилл:** session-change-documenter

## Затронутая секция

- **`Ключевые возможности`** — добавлен глобальный режим `/all`
- **`Структура проекта`** — добавлен модуль `src/claude_manager/all_projects_monitor.py`
- **`Архитектурные принципы`** — уточнена зона ответственности мониторинга
- **`Важные детали для разработки`** — добавлен контракт global all-mode

## Было

`CLAUDE.md` описывал watcher как единый механизм мониторинга сессий и не фиксировал отдельный глобальный монитор для всех проектов.

## Стало

`CLAUDE.md` фиксирует, что `session_watcher` отвечает за активный проект, а `all_projects_monitor` отвечает за режим `/all` по всем проектам, хранит отдельные cursor-счётчики и не помечает сообщения прочитанными для обычной pending-доставки.

## Причина

Режим `/all` стал глобальным: пользователь видит сообщения из всех проектов и переходит к сессии по командам вида `/3s12`. Это меняет быстрый проектный контекст, который будущие сессии читают перед работой.

- **`CLAUDE.md`** — обновлён проектный контекст
- **`src/claude_manager/all_projects_monitor.py`** — новый модуль, который нужно видеть в структуре проекта
- **`dev/docs/adr/14.05_16.55-session-change-documenter-global-all-project-monitoring.md`** — архитектурное решение об отдельном глобальном мониторе
