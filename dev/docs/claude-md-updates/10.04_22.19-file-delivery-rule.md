# CLAUDE.md update: глобальное правило File Delivery Rule

- **Дата:** 10-04-2026
- **Пайплайн:** pipeline-designer (доставка файлов через маркеры)
- **Агент:** agent-documentalist
- **Скилл:** pipeline-designer

## Затронутая секция

Глобальный `~/.claude/CLAUDE.md` — добавлена новая секция **File Delivery Rule**.

## Было

Секции не существовало. Claude во всех проектах не знал про маркеры `[SEND_FILE:path]` и не мог использовать их для доставки файлов пользователю через Telegram-бот.

## Стало

Добавлена секция, которая учит Claude вставлять маркер `[SEND_FILE:/абсолютный/путь]` в текст ответа, когда пользователь просит показать, прислать или сгенерировать файл. Бот-посредник (Telegram) распознаёт маркер и доставляет файл: текстовые рендерятся в чат, бинарные отправляются как вложения.

Правила из секции:
- Путь должен быть абсолютным
- Не использовать Read для показа содержимого файла пользователю — вставлять маркер
- Можно несколько маркеров в одном ответе
- Маркер ставится в любом месте текста — бот вырежет и отправит файл отдельно

## Причина

В проекте claude_manager реализована функциональность доставки файлов из ответа Claude через маркеры `[SEND_FILE:path]` (модуль `file_sender.py`, интеграция в `bot.py`). Чтобы Claude знал про эту возможность и использовал её во всех проектах, добавлено глобальное правило.

## Связанные файлы

- **`~/.claude/CLAUDE.md`** — добавлена секция File Delivery Rule
- **`src/claude_manager/file_sender.py`** — модуль, который обрабатывает маркеры на стороне бота
- **`src/claude_manager/bot.py`** — интеграция file_sender в отправку ответов
