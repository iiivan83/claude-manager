# CLAUDE.md update: process_manager state split

**Дата:** 31-05-2026
**Пайплайн:** session-change-documenter
**Агент:** session-change-documenter (standalone)
**Скилл:** session-change-documenter

## Затронутая секция

`CLAUDE.md` — разделы «Структура проекта», «Слоёная архитектура» и «Состояние в памяти».

## Было

`CLAUDE.md` описывал `process_manager.py` как модуль, который одновременно управляет жизненным циклом процессов и хранит их in-memory state:

```markdown
│   ├── process_manager.py    # Жизненный цикл процессов Claude (ретраи, /stop, прогресс)
```

В архитектурном описании слой инфраструктуры включал только `process_manager` и `claude_runner`, а state процессов был привязан к `process_manager`.

## Стало

В структуру проекта добавлен новый модуль:

```markdown
│   ├── process_manager.py    # Жизненный цикл процессов Claude (запуск, ретраи, /stop, прогресс)
│   ├── process_state.py      # In-memory state процессов CLI: процессы, busy-флаги, stop-events, alias temp→real
```

Слой инфраструктуры теперь явно включает `process_state`, а раздел «Состояние в памяти» фиксирует новую границу:

```markdown
Состояние хранится в словарях на уровне модулей: session_manager (привязки chat_id → session_id), daily_session_registry (дневные номера), process_state (процессы CLI, busy-флаги, stop-events и alias temp→real session_id), session_watcher (счётчики обработанных сообщений активного проекта), all_projects_monitor (чаты в global all, cursor-счётчики по проекту/сессии/backend и link registry для команд `/3s12`).
```

Также добавлено уточнение: `process_manager` остаётся владельцем поведения запуска, retry и `/stop`, но реэкспортирует state-объекты из `process_state` для обратной совместимости.

## Причина

В сессии выполнен первый безопасный разрез большого `process_manager.py`: из него вынесено только конкурентное in-memory state. Это архитектурное изменение нужно зафиксировать в `CLAUDE.md`, потому что документ используется как канонический ориентир по структуре проекта и границам модулей.

- **`src/claude_manager/process_manager.py`** — orchestration-модуль, который теперь импортирует и реэкспортирует state-объекты
