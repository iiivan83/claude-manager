# Changelog: Раннее уведомление о смене session_id (session_id_callback)

## Дата: 12.04.2026

## Тип: Исправление бага (medium)

## Что изменилось

Закрыта гонка (race condition) между обработчиком сообщений и watcher (фоновым наблюдателем за файлами сессий), из-за которой один и тот же UUID регистрировался под двумя дневными номерами. Например, пользователь создавал сессию #20, а watcher через пару секунд регистрировал тот же файл как #21. За 3 дня было зафиксировано 19 таких дублей.

Суть проблемы: при создании новой сессии бот присваивает временный ID (`_new_XXXX`), а Claude CLI создаёт файл с настоящим UUID. Watcher каждые 2 секунды сканирует файлы — и мог увидеть новый UUID раньше, чем бот успевал обновить привязки. В итоге watcher не знал, что этот UUID уже есть под другим ID, и регистрировал дубликат.

Решение: добавлен callback-механизм `session_id_callback` — async-функция, которая вызывается внутри `_process_events()` (функция чтения потока событий от Claude) при первом обнаружении нового session_id. Callback мгновенно обновляет все привязки — `session_watcher`, `session_manager`, `daily_session_registry` — до того, как watcher проснётся для очередного сканирования.

## Затронутые файлы

- **src/claude_manager/process_manager.py** — добавлен тип `SessionIdCallback`. Параметр `session_id_callback` пробрасывается через всю цепочку вызовов: `send_message` → `_execute_send` → `_retry_loop` → `_execute_single_retry` → `_process_events`. В `_process_events` callback вызывается ровно один раз (защита флагом `callback_fired`), обёрнут в try/except (чтобы ошибка в callback не прервала чтение ответа Claude)
- **src/claude_manager/bot.py** — в `_send_to_claude_and_respond()` создано замыкание `_on_session_id_changed`, которое мгновенно обновляет `session_watcher`, `session_manager` и локальную переменную `session_id` через `nonlocal`. Упрощён `_handle_claude_result()` — убран дублирующий блок обновления session_id (теперь через callback)
- **tests/test_process_manager.py** — 6 новых тестов: однократный вызов callback, обработка ошибок, обратная совместимость при `callback=None`
- **tests/test_bot.py** — 4 новых теста в классе `TestSessionIdCallback`: проверка передачи callback, обновление всех модулей, корректный resume с новым ID
