# Changelog — апрель 2026

### 11.04.2026
- **fix(process_manager):** добавлен `asyncio.Lock` (механизм, который не даёт двум параллельным задачам одновременно менять одни и те же данные) для защиты `_busy_flags` — словаря, который отслеживает, занята ли сессия обработкой запроса. Раньше два одновременных запроса к одной сессии могли оба пройти проверку занятости и запуститься параллельно, теперь это невозможно. Lock захватывается только на короткие критические секции (проверка и установка busy-флага, очистка, перенос ключей), чтобы не блокировать длительные операции вроде отправки сообщений в Claude.
