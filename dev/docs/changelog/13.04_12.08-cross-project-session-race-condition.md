# Changelog: Исправление гонки при переключении проектов

## Дата: 13.04.2026

## Тип: Исправление бага (medium)

## Что изменилось

Закрыто окно гонки (race condition) при переключении проектов через `/pN`. Суть проблемы: watcher (фоновый процесс, который каждые 2 секунды проверяет файлы сессий) мог проснуться в момент, когда `config.WORKING_DIR` уже указывал на новый проект, а дневной реестр ещё хранил данные от старого. В итоге сессии из нового проект попадали в `daily_sessions.json` старого проекта — «призрачные» записи.

За 3 дня (11-13 апреля) было зафиксировано 6 таких случаев. Побочный эффект — 32 954 повторных WARNING-сообщения «Файл сессии не найден» в логах (9.5 МБ), потому что watcher пытался прочитать несуществующие файлы каждые 2 секунды.

### Три независимых исправления

- **Глобальная пауза watcher** — добавлены функции `pause_all()` и `resume_all()` в `session_watcher` (модуль мониторинга сессий). Перед сменой `config.WORKING_DIR` вызывается `pause_all()`, после полного сброса всех state-модулей — `resume_all()`. Блок обёрнут в try/finally, чтобы watcher возобновился даже при ошибке. Пока пауза активна, `_poll_sessions()` делает ранний return — ни одного опроса не выполняется
- **Кросс-проверка записей реестра** — новая функция `_remove_orphan_entries()` в `daily_session_registry` (модуль дневной нумерации сессий). При загрузке реестра из файла проверяет каждую запись: существует ли `.jsonl` файл на диске. Записи без файлов удаляются автоматически — это очищает «призрачные» записи, накопившиеся до исправления. Записи с временным ID (`_new_*`) пропускаются — для них файлов нет по определению
- **Защита от спама логов** — множество `_missing_file_sessions` в `session_watcher` запоминает сессии, для которых файл не найден. Повторный WARNING не генерируется — одно предупреждение вместо тысяч

### Обновление CLAUDE.md

- Добавлен новый архитектурный принцип «Атомарность переключения проекта» — описывает обязательный порядок: pause → switch → reset → resume
- Обновлено описание процесса переключения в разделе «Важные детали» — теперь отражает реальную последовательность операций с глобальной паузой и кросс-проверкой файлов

## Затронутые файлы

- **src/claude_manager/session_watcher.py** — новые переменные `_global_paused` и `_missing_file_sessions`, функции `pause_all()` и `resume_all()`, ранний return в `_poll_sessions()` при глобальной паузе, проверка `_missing_file_sessions` в `_check_session()`, очистка новых переменных в `reset_state()` и `start()`
- **src/claude_manager/project_manager.py** — в `_perform_switch()` добавлен вызов `pause_all()` перед сменой WORKING_DIR и `resume_all()` в finally-блоке после `_reset_all_state_modules()`
- **src/claude_manager/daily_session_registry.py** — новая функция `_remove_orphan_entries()`, вызов в `load_registry()` после очистки фантомов и дубликатов, единое сохранение очищенного реестра
- **CLAUDE.md** — новый принцип «Атомарность переключения проекта», обновлённое описание переключения проектов
- **tests/test_session_watcher.py** — тесты на `pause_all`/`resume_all`, `_missing_file_sessions`, очистку при `reset_state()` и `start()`
- **tests/test_project_manager.py** — тесты на вызов `pause_all`/`resume_all` при переключении, try/finally семантику
- **tests/test_daily_session_registry.py** — тесты на `_remove_orphan_entries()`, пропуск `_new_*`, единое сохранение
- **tests/integration/test_project_switching.py** — интеграционные тесты на координацию pause/resume при переключении
