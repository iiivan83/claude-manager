# Changelog: Отправка файлов через telegramify-markdown

## Дата: 12.04.2026

## Тип: Новая функциональность (medium)

## Что изменилось

Бот теперь умеет отправлять файлы из ответа Claude прямо в Telegram. Когда Claude хочет показать файл пользователю, он вставляет в текст маркер `[SEND_FILE:/path/to/file]`. Бот находит эти маркеры, определяет тип файла и отправляет:

- **Текстовые файлы** (`.md`, `.py`, `.json`, `.sh` и ещё ~30 расширений) — рендерятся через `telegramify-markdown` (библиотека для преобразования Markdown в формат Telegram) и отправляются как сообщения с нативным форматированием (entities API, а не parse_mode). Перед содержимым файла идёт заголовок с именем файла
- **Бинарные файлы** (`.png`, `.pdf`, `.zip` и любые другие) — отправляются через `send_document` как вложение
- **Ошибки** (файл не найден, нет доступа, слишком большой) — пользователь получает понятное сообщение, бот не падает

Маркеры обрабатываются только в финальных ответах (`is_final=True`). Промежуточные обновления (thinking от Claude) маркеры не трогают.

## Затронутые файлы

- **src/claude_manager/file_sender.py** — новый утилитный модуль без состояния. Парсинг маркеров `[SEND_FILE:path]`, определение типа файла (текстовый/бинарный), чтение и рендеринг через `telegramify-markdown`, конвертация entities из формата telegramify-markdown в формат python-telegram-bot, проверка бинарных файлов перед отправкой
- **src/claude_manager/bot.py** — интеграция file_sender. Новые приватные функции: `_process_file_markers()` (общая точка входа для send_response и send_watcher_message), `_send_text_file()`, `_send_binary_file()`, `_shift_entity()`. Вызов `_process_file_markers` добавлен в `send_response` и `send_watcher_message` перед `message_splitter.prepare_message`
- **requirements.txt** — добавлена зависимость `telegramify-markdown>=1.1.2`
- **pyproject.toml** — добавлена зависимость `telegramify-markdown>=1.1.2`
- **tests/test_file_sender.py** — новый файл, юнит-тесты для всех публичных функций file_sender
- **tests/test_bot.py** — новые тесты для интеграции file_sender в send_response и send_watcher_message
- **tests/integration/test_message_path.py** — интеграционный тест: полный путь сообщения с маркером [SEND_FILE:...] от Claude до Telegram
