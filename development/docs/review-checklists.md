# Чеклисты для код-ревью

Этот документ содержит чеклисты для 4-проходного ревью кода проекта. Используется скиллом review-code.

## Проход 1: Качество кода

- [ ] Все имена переменных/функций понятны без комментариев
- [ ] Нет однобуквенных переменных (кроме `i` в простых циклах)
- [ ] Все функции до 20 строк
- [ ] Максимум 2 уровня вложенности (if/for внутри if/for)
- [ ] Нет magic numbers (необъяснённых чисел/строк в коде)
- [ ] Нет дублирования кода
- [ ] Type hints на всех публичных функциях
- [ ] Нет типа `any`
- [ ] Docstrings на публичных функциях (одна строка — что делает)
- [ ] Нет сокращений кроме общеизвестных (url, id — можно; col, btn, evt — нельзя)

## Проход 2: Безопасность

- [ ] Токены/секреты только в `.env`, никогда в коде
- [ ] Белый список пользователей (ALLOWED_USER_IDS) проверяется при каждом запросе
- [ ] HTML-экранирование пользовательских данных через `html.escape()`
- [ ] Нет command injection (подстановки пользовательского ввода в shell-команды)
- [ ] Файловые пути валидируются (нет path traversal)
- [ ] Нет захардкоженных паролей или API-ключей
- [ ] CORS, порты и URL настраиваемые через переменные окружения

## Проход 3: Архитектура

- [ ] Транспортный слой (bot.py) не знает про процессы Claude
- [ ] Бизнес-логика (session_manager, daily_session_registry) не импортирует Telegram или процессы
- [ ] Инфраструктура (process_manager, claude_runner) не импортирует бизнес-логику
- [ ] Все I/O операции — async/await
- [ ] Файловые операции через `asyncio.to_thread()`
- [ ] Нет `time.sleep()` — только `asyncio.sleep()`
- [ ] Атомарные операции с файлами (запись через `.tmp` + rename)
- [ ] Нет deadlocks (взаимных блокировок)
- [ ] Функции либо возвращают значение, ЛИБО меняют состояние (не оба)
- [ ] Верхний слой вызывает нижний, но не наоборот

## Проход 4: Соответствие BRD

- [ ] Каждый CJM из BRD реализован
- [ ] Каждый пункт «что может пойти не так» обработан в коде
- [ ] Карта состояний пользователя (3 состояния) работает по описанию BRD
- [ ] Проверка доступа по белому списку — при каждом действии
- [ ] Промежуточные обновления (прогресс) — не чаще раза в 30 сек, с номером сессии
- [ ] Координация мониторинга и ответов — нет дублей
- [ ] Одновременная работа Telegram + терминал работает
- [ ] Защита от двойного запуска бота (файл-замок + Conflict)
- [ ] Восстановление состояния при перезапуске
- [ ] Дневная нумерация сессий (#1, #2, #3...)
