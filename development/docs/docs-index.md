# Индекс документации проекта

Справочник по всем документам проекта Claude Manager: что в каждом документе и когда его читать.

## CLAUDE.md
Читать если:
- Первоначальное знакомство с проектом
- Нужны стандарты кода, команды запуска или структура проекта
- Нужны правила кодовой базы и архитектурные принципы

## development/docs/brd-user-journeys.md
BRD с описанием всех пользовательских путей (CJM) через бота.

Читать если:
- Нужно понять все сценарии взаимодействия пользователя с ботом
- Нужно проверить ожидаемое поведение конкретной команды или процесса
- Нужно понять переходы между состояниями пользователя (нет сессии / подключён / мониторинг)
- Нужно найти edge cases для конкретного сценария

## development/docs/deployment-guide.md
Пошаговая инструкция по запуску бота с чистой системы.

Читать если:
- Нужно развернуть бота на новой машине
- Нужно вспомнить порядок настройки .env
- Возникла проблема при запуске бота (раздел «Решение проблем»)

## development/docs/review-checklists.md
Чеклисты для 4-проходного код-ревью (качество, безопасность, архитектура, BRD).

Читать если:
- Нужно провести ревью кода проекта
- Нужно проверить соответствие кода стандартам качества

## development/docs/review-report_30-03.md
Отчёт код-ревью с описанием найденных и исправленных проблем.

Читать если:
- Нужно понять какие проблемы были найдены и исправлены при ревью
- Нужен контекст для понимания текущего состояния кода

## development/docs/brd-validation-report_29-03-23-47.md
Отчёт валидации BRD: проверка документации на противоречия и неточности.

Читать если:
- Нужно понять какие проблемы были найдены при валидации BRD
- Нужен контекст принятых решений по спорным вопросам

## development/docs/claude-cli-stream-json-protocol.md
Полный справочник по протоколу обмена данными с Claude Code CLI через stdin/stdout.

Читать если:
- Нужно понять формат JSON-сообщений для отправки в Claude CLI
- Нужно понять типы событий, которые приходят от Claude CLI
- Нужно работать с control protocol (управление процессом)
- Нужно узнать известные баги протокола

## development/docs/root-cause-reports/
Отчёты анализа первопричин проблем. Содержат цепочку причин, рекомендации и чек-листы исправлений.

Читать если:
- Нужно понять корневую причину конкретной проблемы
- Нужно проверить рекомендации по предотвращению

## development/docs/session-reports/
Отчёты о проделанной работе за сессию. Организованы по дате в подпапках `DD-MM/`.

Читать если:
- Нужно понять что делалось в предыдущих сессиях
- Нужен контекст для продолжения работы
- Нужно найти когда и почему было принято определённое решение

## development/specs/pipeline-spec.md
Спецификация инженерного пайплайна: инструментарий (скиллы, документы, скрипты) + процесс (оркестратор).

Читать если:
- Нужно создать скиллы пайплайна (промпты для skill-creator внутри)
- Нужно понять порядок реализации проекта от BRD до готового кода
- Нужно понять ворота качества между фазами
- Нужно понять как работает оркестратор pipeline-run

## development/specs/module-dependency-graph.md
Граф зависимостей между модулями проекта.

Читать если:
- Нужно понять какой модуль от какого зависит
- Нужно проверить архитектурные границы между слоями

## development/script-specs/
Спецификации (ТЗ) на вспомогательные скрипты разработки. Реализованные спецификации перемещаются в `realized/`.

Читать если:
- Нужно понять что должен делать конкретный скрипт
- Нужно реализовать скрипт по спецификации
- Нужно проверить требования к скрипту перед реализацией

## development/scripts/
Готовые вспомогательные скрипты разработки (утилиты, анализаторы, генераторы).

## development/temp-docs/
Временные рабочие файлы: промежуточные результаты анализа, черновики, промпты.

## watch_and_restart.sh
Скрипт для автоматического перезапуска бота при изменении файлов.
