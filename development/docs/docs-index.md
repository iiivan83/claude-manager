# Индекс документации проекта

Справочник по всем документам проекта Claude Manager: что в каждом документе и когда его читать.

## CLAUDE.md
Читать если:
- Первоначальное знакомство с проектом
- Нужны стандарты кода, команды запуска или структура проекта
- Нужны правила кодовой базы и архитектурные принципы

## development/docs/brd-user-journeys.md
BRD с описанием всех пользовательских путей (CJM) через бота.

Читать если:
- Нужно понять все сценарии взаимодействия пользователя с ботом
- Нужно проверить ожидаемое поведение конкретной команды или процесса
- Нужно понять переходы между состояниями пользователя (нет сессии / подключён / мониторинг)
- Нужно найти edge cases для конкретного сценария

## development/docs/session-reports/
Отчёты о проделанной работе за сессию. Организованы по дате в подпапках `DD-MM/`.

Читать если:
- Нужно понять что делалось в предыдущих сессиях
- Нужен контекст для продолжения работы
- Нужно найти когда и почему было принято определённое решение

## development/specs/pipeline-spec.md
Спецификация инженерного пайплайна: инструментарий (скиллы, документы, скрипты) + процесс (оркестратор).

Читать если:
- Нужно создать скиллы пайплайна (промпты для skill-creator внутри)
- Нужно понять порядок реализации проекта от BRD до готового кода
- Нужно понять ворота качества между фазами
- Нужно понять как работает оркестратор pipeline-run

## development/script-specs/
Спецификации (ТЗ) на вспомогательные скрипты разработки. Реализованные спецификации перемещаются в `realized/`.

Читать если:
- Нужно понять что должен делать конкретный скрипт
- Нужно реализовать скрипт по спецификации
- Нужно проверить требования к скрипту перед реализацией

## development/scripts/
Готовые вспомогательные скрипты разработки (утилиты, анализаторы, генераторы).

## development/temp-docs/
Временные рабочие файлы: промежуточные результаты анализа, черновики, промпты.

## watch_and_restart.sh
Скрипт для автоматического перезапуска бота при изменении файлов.
