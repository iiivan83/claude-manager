# Спецификация модуля: message_splitter

Дата: 30-03-2026
Слой: 0 (без зависимостей от других модулей проекта)
Файл: `src/claude_manager/message_splitter.py`

## Назначение

Конвертирует ответы Claude из Markdown в HTML-формат Telegram и разбивает длинные сообщения на части до 4096 символов с корректной починкой HTML-тегов на границах. Модуль работает без состояния (stateless) — чистые функции без побочных эффектов.

## Обслуживаемые сценарии

- **CJM-02: Отправка текстового сообщения** — конвертация ответа Claude (Markdown) в HTML для Telegram, разбивка длинных ответов на части до 4096 символов с починкой тегов, подготовка текста для отправки
- **CJM-03: Отправка фотографии или файла** — та же конвертация и разбивка для ответов Claude на фото/файлы
- **Сквозной механизм «Промежуточные обновления»** — форматирование промежуточных рассуждений Claude (текст курсивом) для отправки в Telegram
- **Сквозной механизм «Отправка сообщений в Telegram»** — вся цепочка: конвертация Markdown в HTML, HTML-экранирование, разбивка, починка тегов, fallback на plain text

## Публичный API

### `markdown_to_html(markdown_text: str) -> str`

Конвертирует текст из Markdown-формата (в котором отвечает Claude) в HTML-формат, который понимает Telegram.

**Аргументы:**
- `markdown_text` (str) — текст в формате Markdown от Claude

**Возвращает:** строку в HTML-формате Telegram (теги `<b>`, `<i>`, `<code>`, `<pre>`, `<a>`)

**Исключения:** не выбрасывает. При невозможности распознать Markdown-элемент — оставляет текст как есть.

---

### `split_html_message(html_text: str) -> list[str]`

Разбивает длинное HTML-сообщение на части, каждая из которых не длиннее 4096 символов. При разбивке починяет HTML-теги: если тег открылся в одной части, но текст продолжается — тег закрывается в конце этой части и открывается заново в начале следующей.

**Аргументы:**
- `html_text` (str) — HTML-текст, который нужно разбить (уже сконвертированный из Markdown)

**Возвращает:** список строк, каждая не длиннее `TELEGRAM_MESSAGE_LIMIT` (4096) символов. Если исходный текст короче лимита — список из одного элемента.

**Исключения:** не выбрасывает. Даже при некорректном HTML возвращает разбитый текст (best effort).

---

### `strip_html_tags(html_text: str) -> str`

Удаляет все HTML-теги из текста, оставляя только содержимое. Используется как fallback (запасной вариант) — когда Telegram не смог распарсить HTML-форматирование, бот отправляет обычный текст без тегов.

**Аргументы:**
- `html_text` (str) — HTML-текст с тегами

**Возвращает:** строку без HTML-тегов. HTML-сущности (`&amp;`, `&lt;`, `&gt;`) декодируются обратно в символы (`&`, `<`, `>`).

**Исключения:** не выбрасывает.

---

### `prepare_message(markdown_text: str) -> list[str]`

Комбинированная функция: конвертирует Markdown в HTML и разбивает на части. Это основная точка входа для модуля `bot` — одним вызовом из Markdown-ответа Claude получаются готовые к отправке части.

**Аргументы:**
- `markdown_text` (str) — текст ответа Claude в формате Markdown

**Возвращает:** список строк в HTML-формате, каждая не длиннее 4096 символов. Готовы к отправке в Telegram с `parse_mode="HTML"`.

**Исключения:** не выбрасывает.

## Внутренние функции

### `_escape_html(text: str) -> str`

Экранирует спецсимволы HTML в обычном тексте (вне тегов), чтобы Telegram не интерпретировал их как разметку.

**Аргументы:**
- `text` (str) — фрагмент обычного текста (не внутри HTML-тега)

**Возвращает:** строку с заменёнными символами: `&` -> `&amp;`, `<` -> `&lt;`, `>` -> `&gt;`

**Примечание:** Экранирование увеличивает длину текста — символ `&` (1 символ) превращается в `&amp;` (5 символов). Именно поэтому длина при разбивке считается после экранирования.

---

### `_parse_markdown_blocks(markdown_text: str) -> list[dict[str, str]]`

Разбирает Markdown-текст на структурные блоки (параграфы, блоки кода, заголовки, списки). Каждый блок — словарь с типом и содержимым.

**Аргументы:**
- `markdown_text` (str) — исходный Markdown-текст

**Возвращает:** список словарей вида `{"type": "paragraph" | "code_block" | "heading" | "list_item", "content": "...", "language": "python"}`. Поле `language` присутствует только для `code_block`.

---

### `_convert_inline_markdown(text: str) -> str`

Конвертирует inline-элементы Markdown в HTML-теги Telegram: жирный, курсив, код, ссылки.

**Аргументы:**
- `text` (str) — текст с inline-Markdown (звёздочки, обратные кавычки, квадратные скобки)

**Возвращает:** строку с HTML-тегами.

**Преобразования:**
- `**text**` -> `<b>text</b>` (жирный)
- `*text*` или `_text_` -> `<i>text</i>` (курсив)
- `` `code` `` -> `<code>code</code>` (inline-код)
- `[text](url)` -> `<a href="url">text</a>` (ссылка)

---

### `_detect_nested_code_blocks(text: str) -> list[tuple[int, int, bool]]`

Находит все блоки кода (тройные обратные кавычки ```) в тексте, включая вложенные — когда Claude показывает пример Markdown с тройными кавычками внутри блока кода.

**Аргументы:**
- `text` (str) — весь Markdown-текст

**Возвращает:** список кортежей `(start_pos, end_pos, is_outer)`, где `is_outer` указывает, является ли блок внешним (настоящим) блоком кода или вложенным (примером).

---

### `_find_split_point(html_text: str, max_length: int) -> int`

Находит оптимальную точку разбивки в HTML-тексте — позицию, в которой можно разрезать сообщение, не ломая HTML-теги и слова.

**Аргументы:**
- `html_text` (str) — HTML-текст, который нужно разрезать
- `max_length` (int) — максимальная длина части (с учётом символов на закрытие/открытие тегов)

**Возвращает:** индекс символа, на котором нужно разрезать текст.

**Приоритет точек разбивки (от лучшей к худшей):**
1. Между блоками `</pre>\n` — между блоками кода или между блоком и параграфом
2. Двойной перенос строки `\n\n` — граница параграфа
3. Одиночный перенос строки `\n` — конец строки
4. Пробел ` ` — граница слова
5. Любая позиция — крайний случай, если ничего лучше не нашлось

---

### `_collect_open_tags(html_text: str) -> list[tuple[str, str]]`

Анализирует фрагмент HTML и определяет, какие теги остались незакрытыми на конце фрагмента. Хранит и имя тега, и полную открывающую строку (с атрибутами) — это нужно для корректного повторного открытия тегов вроде `<code class="language-python">`.

**Аргументы:**
- `html_text` (str) — фрагмент HTML-текста

**Возвращает:** список кортежей `(tag_name, full_opening_tag)` в порядке открытия. Например, для текста `<pre><code class="language-python">print(1)` вернёт `[("pre", "<pre>"), ("code", '<code class="language-python">')]`.

---

### `_close_tags(open_tags: list[tuple[str, str]]) -> str`

Формирует строку с закрывающими тегами — для добавления в конец части сообщения.

**Аргументы:**
- `open_tags` (list[tuple[str, str]]) — список незакрытых тегов (от `_collect_open_tags`)

**Возвращает:** строку с закрывающими тегами в обратном порядке (последний открытый — первый закрытый). Для `[("b", "<b>"), ("i", "<i>")]` вернёт `"</i></b>"`.

---

### `_reopen_tags(open_tags: list[tuple[str, str]]) -> str`

Формирует строку с открывающими тегами — для добавления в начало следующей части сообщения. Использует полную строку тега (с атрибутами), а не только имя — чтобы сохранить атрибуты вроде `class="language-python"`.

**Аргументы:**
- `open_tags` (list[tuple[str, str]]) — список незакрытых тегов (от `_collect_open_tags`)

**Возвращает:** строку с открывающими тегами в прямом порядке. Для `[("pre", "<pre>"), ("code", '<code class="language-python">')]` вернёт `'<pre><code class="language-python">'`.

## Алгоритм работы

### markdown_to_html

1. Вызывает `_parse_markdown_blocks` — разбивает текст на структурные блоки (параграфы, код, заголовки, списки)
2. Для каждого блока применяет преобразование в зависимости от типа:
   - `code_block` — оборачивает в `<pre><code class="language-{lang}">...</code></pre>`, содержимое экранирует через `_escape_html` (чтобы код внутри `<pre>` отображался как есть, а не как HTML)
   - `heading` — оборачивает в `<b>...</b>` (Telegram не поддерживает тег `<h1>`)
   - `list_item` — для маркированных списков (`- `, `* `) заменяет маркер на `  ` (два пробела как отступ). Для нумерованных (`1. `, `2. `) оставляет номер как есть. Inline-Markdown внутри пунктов обрабатывается через `_convert_inline_markdown`. Telegram не поддерживает `<ul>/<li>`
   - `paragraph` — вызывает `_convert_inline_markdown` для обработки жирного, курсива, кода и ссылок. Обычный текст (вне inline-элементов) экранирует через `_escape_html`
3. Собирает все блоки в единую HTML-строку, разделяя двойным переносом строки `\n\n`
4. Возвращает готовый HTML

### split_html_message

1. Проверяет длину `html_text` — если `<= TELEGRAM_MESSAGE_LIMIT`, возвращает `[html_text]`
2. Рассчитывает доступную длину для текста в текущей части: `TELEGRAM_MESSAGE_LIMIT` минус запас на закрывающие/открывающие теги (оценивается как `MAX_TAG_OVERHEAD_PER_SPLIT` = 100 символов)
3. Вызывает `_find_split_point` для поиска оптимальной точки разрезки в пределах доступной длины
4. Вызывает `_collect_open_tags` для фрагмента до точки разрезки — определяет, какие теги незакрыты
5. Добавляет в конец первой части закрывающие теги (`_close_tags`)
6. Начинает вторую часть с открывающих тегов (`_reopen_tags`)
7. Рекурсивно повторяет шаги 2-6 для оставшегося текста (со вставленными открывающими тегами), пока длина оставшегося текста не станет `<= TELEGRAM_MESSAGE_LIMIT`
8. Возвращает список всех частей

### strip_html_tags

1. Удаляет все HTML-теги регулярным выражением `<[^>]+>` — заменяет на пустую строку
2. Декодирует HTML-сущности: `&amp;` -> `&`, `&lt;` -> `<`, `&gt;` -> `>`, `&quot;` -> `"`, `&#x27;` -> `'`
3. Возвращает очищенный текст

### prepare_message

1. Вызывает `markdown_to_html(markdown_text)` — конвертация Markdown в HTML
2. Вызывает `split_html_message(html_result)` — разбивка на части
3. Возвращает результат

### _parse_markdown_blocks

1. Вызывает `_detect_nested_code_blocks` для нахождения всех блоков кода с учётом вложенности
2. Проходит по тексту построчно, отслеживая текущий контекст (внутри блока кода или нет)
3. Для каждого блока кода (отмеченного как `is_outer=True`) создаёт блок типа `code_block` с извлечением языка из строки открытия (```python -> language="python")
4. Текст вне блоков кода разбивает на параграфы по двойному переносу строки
5. Внутри каждого параграфа определяет заголовки (строки начинающиеся с `#`, `##`, `###`) и пункты списков (строки начинающиеся с `- `, `* `, `1. `)
6. Возвращает упорядоченный список блоков

### _detect_nested_code_blocks

1. Ищет все вхождения ` ``` ` в тексте с помощью поиска подстроки
2. Строит стек: каждое вхождение ` ``` ` либо открывает блок, либо закрывает текущий открытый
3. Первый уровень стека — настоящие (внешние) блоки кода. Вложенные ` ``` ` внутри открытого блока — это примеры, не настоящие границы
4. Возвращает список всех найденных блоков с позициями и флагом `is_outer`

### _find_split_point

1. Ограничивает область поиска: от `max_length * 0.5` (не разрезать слишком рано) до `max_length`
2. Ищет в этой области точки разбивки по приоритету:
   - Сначала ищет `</pre>\n` — граница между блоками кода
   - Затем `\n\n` — граница параграфа
   - Затем `\n` — конец строки
   - Затем ` ` — пробел
3. Если ничего не нашлось — разрезает ровно на `max_length`
4. При поиске проверяет, что точка разрезки не попадает внутрь HTML-тега (между `<` и `>`)
5. Возвращает индекс точки разрезки

### _collect_open_tags

1. Находит все HTML-теги в тексте регулярным выражением `<(/?)(\w+)([^>]*)>`, захватывая три группы: слеш (если закрывающий), имя тега, атрибуты
2. Ведёт стек кортежей `(tag_name, full_opening_tag)`: открывающий тег добавляет `(name, full_match)` в стек, закрывающий — убирает последний кортеж с тем же `tag_name`
3. Учитывает самозакрывающиеся теги (`<br/>`) — они не добавляются в стек
4. Фильтрует по `SUPPORTED_TAGS` — отслеживает только теги, которые поддерживает Telegram (`b`, `i`, `code`, `pre`, `a`)
5. Возвращает содержимое стека — это незакрытые теги с их полными открывающими строками

## Обработка ошибок

Модуль `message_splitter` — чистый трансформатор текста. Он не работает с файлами, сетью или внешними системами, поэтому не выбрасывает исключений.

- **Некорректный Markdown (незакрытые кавычки, битые ссылки)** — функция `markdown_to_html` оставляет нераспознанные элементы как обычный текст. Не ломается, не выбрасывает ошибку.
- **Некорректный HTML (незакрытые теги)** — функция `split_html_message` работает в режиме «best effort»: собирает информацию о незакрытых тегах по тому, что есть. Если тег оказался незакрытым — закроет его в конце части.
- **Пустая строка на входе** — все функции корректно обрабатывают пустую строку: `markdown_to_html("")` -> `""`, `split_html_message("")` -> `[""]`, `prepare_message("")` -> `[""]`.
- **Текст состоит только из спецсимволов HTML** — `_escape_html` корректно экранирует все спецсимволы, длина увеличивается. `split_html_message` учитывает увеличенную длину.
- **Очень длинная строка без пробелов и переносов** — `_find_split_point` использует fallback на позицию `max_length`, разрезая строку принудительно.
- **Глубоко вложенные теги (например, `<b><i><code>...</code></i></b>`)** — `_collect_open_tags` корректно отслеживает стек любой глубины. `_close_tags` закрывает в обратном порядке, `_reopen_tags` открывает в прямом.
- **Блок кода с тройными кавычками внутри** — `_detect_nested_code_blocks` различает внешние (настоящие) и вложенные (примеры) кавычки, не путает уровни.

## Константы

- `TELEGRAM_MESSAGE_LIMIT = 4096` — максимальная длина одного сообщения в Telegram. Определено самим Telegram API, уменьшать нельзя.
- `MAX_TAG_OVERHEAD_PER_SPLIT = 100` — запас символов на закрывающие и открывающие теги при разбивке. Реальные HTML-теги Telegram короткие (`<b>`, `<pre>`, `<code>`), так что 100 символов — с большим запасом. Например, `</code></pre></b></i>` + `<i><b><pre><code>` — это 44 символа.
- `SPLIT_SEARCH_RATIO = 0.5` — при поиске точки разбивки не искать раньше чем на середине доступной длины. Чтобы части были примерно одинаковые, а не крошечная первая и огромная вторая.
- `CODE_BLOCK_DELIMITER = "```"` — маркер начала/конца блока кода в Markdown. Три обратных кавычки.
- `SUPPORTED_TAGS = ["b", "i", "code", "pre", "a"]` — HTML-теги, которые поддерживает Telegram Bot API в режиме HTML. Только эти теги отслеживаются при починке на границах разбивки.

### Граничные случаи

- **test_empty_string** — пустая строка на входе
  - Вход: `""`
  - Ожидаемый результат: `markdown_to_html("")` -> `""`, `split_html_message("")` -> `[""]`, `prepare_message("")` -> `[""]`
  - Тип: edge case

- **test_exactly_4096_chars** — текст ровно 4096 символов (граница лимита)
  - Вход: HTML-текст длиной ровно 4096 символов
  - Ожидаемый результат: `[исходный текст]` — не разбивается
  - Тип: edge case

- **test_4097_chars** — текст на 1 символ длиннее лимита
  - Вход: HTML-текст длиной 4097 символов
  - Ожидаемый результат: 2 части
  - Тип: edge case

- **test_html_escape_increases_length** — экранирование делает текст длиннее лимита
  - Вход: Markdown-текст с 2000 символов `&`, который после экранирования (`&amp;`) станет 10000 символов
  - Ожидаемый результат: разбивка учитывает экранированную длину, каждая часть <= 4096
  - Тип: edge case

- **test_nested_code_block_triple_backticks** — тройные кавычки внутри блока кода

  - Тип: edge case

- **test_no_space_or_newline_for_split** — длинная строка без пробелов и переносов
  - Вход: `"a" * 5000` (5000 символов без пробелов)

  - Тип: edge case

- **test_split_does_not_break_inside_html_tag** — разбивка не попадает внутрь тега
