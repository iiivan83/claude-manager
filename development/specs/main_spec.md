# Спецификация модуля: main

Дата: 30-03-2026
Слой: 5 (верхний, зависит от слоёв 0-4)
Файл: `src/claude_manager/main.py`

## Назначение

Точка входа приложения. Запускает бота в правильном порядке: загружает настройки, защищает от двойного запуска через файл-замок, восстанавливает сохранённое состояние сессий, запускает Telegram-опрос и корректно завершается при остановке.

## Обслуживаемые сценарии

- **CJM-01: Первый запуск и настройка** — загрузка конфигурации из `.env`, проверка обязательных параметров, настройка логирования, восстановление состояния из `sessions.json`, автоматический переход в режим `/all` при отсутствии привязки, запуск Telegram-опроса с пропуском старых сообщений, настройка меню команд
- **CJM-09: Защита от двойного запуска** — захват файла-замка `bot.pid` через `fcntl.flock`, запись PID (идентификатора процесса) в файл, обработка ошибки Conflict от Telegram (два бота с одним токеном)

## Публичный API

### `def main() -> None`

Главная функция приложения — точка входа. Вызывается из `__main__.py`. Выполняет всю последовательность запуска: настройка логирования, загрузка конфигурации, захват файла-замка, восстановление состояния, запуск Telegram-бота. Блокирует поток выполнения до остановки бота.

**Аргументы:** нет.

**Возвращает:** ничего. Функция блокирует выполнение, пока бот работает. При завершении (Ctrl+C, сигнал SIGTERM, ошибка) — функция заканчивается и программа останавливается.

**Исключения:**
- Не выбрасывает — все исключения ловятся внутри, логируются и приводят к завершению программы с кодом ошибки

## Внутренние функции

### `def _setup_logging() -> None`

Настраивает систему логирования (записи событий для отладки) для всего приложения.

**Аргументы:** нет.

**Возвращает:** ничего.

### `def _acquire_lock() -> io.TextIOWrapper | None`

Пытается захватить файл-замок `bot.pid`, чтобы предотвратить запуск второй копии бота. Если замок получен — записывает PID (номер процесса) в файл.

**Аргументы:** нет.

**Возвращает:** открытый файловый объект, если замок успешно захвачен. `None` — если другой экземпляр бота уже работает (замок занят).

**Важно:** файловый объект должен оставаться открытым всё время работы бота. Если файл закрыть — замок освобождается и другой экземпляр сможет запуститься.

### `async def _restore_state() -> None`

Восстанавливает состояние бота после перезапуска: загружает привязки чатов к сессиям из `sessions.json` и дневной реестр нумерации из `daily_sessions.json`.

**Аргументы:** нет.

**Возвращает:** ничего.

### `async def _run_bot() -> None`

Создаёт экземпляр Telegram-бота, регистрирует обработчики команд, настраивает меню команд в Telegram и запускает polling (опрос Telegram на новые сообщения). Обрабатывает ошибку Conflict (два бота с одним токеном).

**Аргументы:** нет.

**Возвращает:** ничего. Блокирует выполнение до остановки бота.

**Исключения:**
- `telegram.error.Conflict` — два бота с одним токеном одновременно опрашивают Telegram. Ловится, логируется и приводит к завершению

## Алгоритм работы

### main

1. **Настроить логирование** — вызвать `_setup_logging()`. Это делается первым, чтобы все дальнейшие события записывались в лог
2. **Загрузить конфигурацию** — вызвать `config.load_config()`. Если настройки неполные (нет токена или списка пользователей) — `ConfigError` ловится, сообщение об ошибке выводится в лог (уровень error), программа завершается с кодом 1
3. **Захватить файл-замок** — вызвать `_acquire_lock()`. Если вернулось None (замок занят другим экземпляром) — залогировать ошибку `"Бот уже запущен (файл bot.pid заблокирован другим процессом)"`, завершиться с кодом 1
4. **Залогировать запуск** — записать в лог (уровень info): `"Claude Manager запускается..."`, указать рабочую директорию из config
5. **Запустить асинхронную часть** — вызвать `asyncio.run(_run_bot())`. Все дальнейшие операции выполняются внутри asyncio event loop (цикла обработки асинхронных задач)
6. **Обработать завершение** — при завершении (любом) файл-замок автоматически освобождается. Залогировать `"Claude Manager остановлен"`

### _setup_logging

1. **Настроить формат** — вызвать `logging.basicConfig()` с параметрами:
   - `level=logging.INFO` — записывать события от уровня INFO и выше
   - `format="%(asctime)s [%(levelname)s] %(name)s: %(message)s"` — формат: время, уровень, имя модуля, сообщение
   - `datefmt="%Y-%m-%d %H:%M:%S"` — формат даты без миллисекунд, для читаемости
2. **Приглушить логи библиотек** — установить уровень WARNING для логгеров `httpx` и `telegram` (библиотеки Telegram-бота), чтобы их подробные отладочные сообщения не засоряли лог

### _acquire_lock

1. **Определить путь к файлу-замку** — `bot.pid` в рабочей директории проекта (`config.WORKING_DIR / "bot.pid"`). Файл создаётся рядом с проектом, а не в системной папке — это проще для пользователя
2. **Открыть файл** — `open(lock_path, "w")`. Файл создаётся, если не существует
3. **Попытаться захватить замок** — `fcntl.flock(file, fcntl.LOCK_EX | fcntl.LOCK_NB)`. Флаг `LOCK_EX` — эксклюзивный (только один процесс может держать). Флаг `LOCK_NB` — не блокирующий (если замок занят — сразу ошибка, а не ожидание)
4. **Если замок получен** — записать PID (номер текущего процесса) в файл: `file.write(str(os.getpid()))`, сбросить буфер: `file.flush()`. Вернуть файловый объект
5. **Если замок занят** — поймать `BlockingIOError` (ошибка, означающая что файл уже заблокирован другим процессом). Закрыть файл. Вернуть `None`

### _restore_state

1. **Загрузить дневной реестр** — вызвать `daily_session_registry.load_registry()`. Реестр загружает файл `daily_sessions.json` с номерами сессий. Если файл отсутствует — будет создан пустой реестр (штатная ситуация при первом запуске)
2. **Загрузить привязки сессий** — вызвать `session_manager.load_sessions()`. Менеджер загружает файл `sessions.json` с привязками чатов к сессиям. Если файл отсутствует — будет создан пустой словарь
3. **Определить начальное состояние** — проверить, есть ли восстановленная привязка к сессии. Если привязки нет — автоматически установить режим `/all` (мониторинг всех сессий) через `session_manager.set_all_mode()`. Это решение из валидации BRD (проблема 4.2): убрано Состояние 1 «Чистый лист», бот всегда в одном из двух рабочих состояний
4. **Залогировать результат** — записать в лог, было ли восстановлено состояние (какая сессия привязана) или бот перешёл в режим `/all`

### _run_bot

1. **Создать приложение Telegram** — вызвать `telegram.ext.ApplicationBuilder().token(config.BOT_TOKEN).build()`. Это создаёт экземпляр приложения python-telegram-bot, который умеет подключаться к Telegram API
2. **Зарегистрировать обработчики** — вызвать `bot.register_handlers(application)`. Модуль bot добавит обработчики для команд (`/new`, `/sessions`, `/all`, `/stop`, `/N`) и текстовых сообщений. Конкретный API зависит от спецификации модуля bot
3. **Восстановить состояние** — вызвать `await _restore_state()`. Выполняется после создания приложения, но до запуска polling — чтобы к моменту получения первого сообщения состояние было готово
4. **Настроить меню команд** — вызвать `await application.bot.set_my_commands(TELEGRAM_COMMANDS)`. Это устанавливает список команд, который пользователь видит нажав `/` в поле ввода Telegram
5. **Запустить polling** — вызвать `application.run_polling()` с параметром `drop_pending_updates=True`. Параметр `drop_pending_updates` отбрасывает все сообщения, которые пришли пока бот был выключен — бот реагирует только на новые (требование BRD CJM-01). Polling блокирует выполнение до остановки
6. **Обработать Conflict** — обернуть `run_polling()` в try/except для `telegram.error.Conflict`. Эта ошибка возникает, когда два бота с одним токеном одновременно опрашивают Telegram. Залогировать (уровень error): `"Обнаружен конфликт: другой бот уже использует этот токен. Завершение."`. Это второй уровень защиты от двойного запуска (CJM-09)
7. **Graceful shutdown** — при получении сигнала завершения (Ctrl+C, SIGTERM) `run_polling()` сам корректно завершается: закрывает соединения, останавливает обработчики. Дополнительной обработки сигналов не нужно — python-telegram-bot делает это автоматически

## Обработка ошибок

- **ConfigError (ошибка конфигурации)** — ловится в `main()`. Сообщение об ошибке записывается в лог (уровень error). Программа завершается с кодом 1 через `sys.exit(1)`. Пользователь видит понятное сообщение в терминале: какой именно параметр отсутствует
- **Файл-замок занят (BlockingIOError)** — `_acquire_lock()` возвращает `None`. В `main()` это обрабатывается как штатная ситуация: `"Бот уже запущен"` записывается в лог, программа завершается с кодом 1. Это защита, а не ошибка
- **Ошибка при восстановлении состояния** — если `sessions.json` или `daily_sessions.json` повреждены — каждый модуль обрабатывает это самостоятельно (создаёт пустое состояние). Модуль main логирует предупреждение, но продолжает работу — бот запустится с чистым состоянием
- **Conflict от Telegram** — два бота с одним токеном. Ловится в `_run_bot()`, логируется как ошибка, программа завершается. Это второй уровень защиты (CJM-09): даже если файл-замок не сработал, Telegram не позволит двум ботам работать одновременно
- **Непредвиденная ошибка** — любое необработанное исключение ловится верхнеуровневым try/except в `main()`. Записывается в лог через `logger.exception()` (с полным стектрейсом — списком вызовов, который показывает где именно произошла ошибка). Программа завершается с кодом 1
- **KeyboardInterrupt (Ctrl+C)** — ловится отдельно от остальных исключений. Это штатное завершение, не ошибка. Логируется на уровне info: `"Получен сигнал завершения"`. Программа завершается с кодом 0 (успех)

## Константы

- `LOCK_FILENAME = "bot.pid"` — имя файла-замка. Выбрано `bot.pid` по конвенции Unix-программ: файл содержит PID (идентификатор процесса) и служит замком
- `TELEGRAM_COMMANDS` — список команд для меню Telegram. Каждая команда — пара (имя, описание):
  - `("new", "Создать новую сессию")`
  - `("sessions", "Показать последние сессии")`
  - `("all", "Режим мониторинга всех сессий")`
  - `("stop", "Остановить Claude")`

## Порядок вызовов при запуске

```
main()
  1. _setup_logging()
  2. config.load_config()              # ← может выбросить ConfigError
  3. _acquire_lock()                   # ← может вернуть None
  4. asyncio.run(_run_bot())
       4a. ApplicationBuilder().build()
       4b. bot.register_handlers(app)
       4c. _restore_state()
             4c-i.   daily_session_registry.load_registry()
             4c-ii.  session_manager.load_sessions()
             4c-iii. session_manager.set_all_mode()  # если нет привязки
       4d. app.bot.set_my_commands(TELEGRAM_COMMANDS)
       4e. app.run_polling(drop_pending_updates=True)
```

## Порядок вызовов при остановке

```
Ctrl+C / SIGTERM
  1. run_polling() получает сигнал и начинает graceful shutdown
  2. Telegram-соединения закрываются
  3. Обработчики останавливаются
  4. _run_bot() завершается
  5. asyncio.run() завершается
  6. main() ловит KeyboardInterrupt (если это был Ctrl+C)
  7. Файл-замок автоматически освобождается (файловый объект закрывается)
  8. Лог: "Claude Manager остановлен"
  9. sys.exit(0)
```
